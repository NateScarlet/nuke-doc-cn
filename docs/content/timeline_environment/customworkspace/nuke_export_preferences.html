核武器/出口偏好

这些首选项定义了 核武器工作室 内部转码或使用外部核弹，以及如何处理转码。

偏好

默认值

描述

核武器路径

依赖于操作系统

当 使用指定的路径 核武器工作室 渲染 启用后，将文件路径设置为您打算在往返过程中使用的外部 Nuke 应用程序，并发送到 Nuke 导出。

注意: 如果您打算使用外部 Nuke 应用程序，Hiero <> Nuke 需要 Nuke 6.3v5 或更高版本才能正确运行。

使用指定的路径 核武器工作室 渲染

残疾人

启用时，中指定的 Nuke 应用程序 核武器路径 用于渲染输出。

禁用时，仅内部命令行 核武器工作室 用于渲染。

始终使用交互式核弹许可证

残疾人

启用后，使用交互式许可证 (如果可用) 在外部核弹上渲染。

作为 NukeX 发射

已启用

启用时，中指定的 Nuke 应用程序 核武器路径 如果有许可证，则作为 NukeX 启动。

注意: 在 Mac 上，此选项呈灰色。如果您需要启动 NukeX，请在 核武器 路径 字段。例如:
/Applications/Nuke8.0v1/NukeX8.0v1.app

核武器限制

限制核武器以提高 核武器工作室 UI 响应

已启用

启用后，用户界面在转码期间仍然响应。

此选项导致 Nuke 在转码期间只使用 2 个线程和 25% 的可用 RAM。因此，转码可能会大大减慢。

不限制核武器

残疾人

启用时，转码使用尽可能多的处理能力，这可能导致用户界面变得无响应。

自定义核武器限制

残疾人

启用后，使用 线程 和 缓存内存 手动配置转码处理的字段:

• 线程 -限制可用于核武器的线程数量。

• 缓存内存 -限制可用于核武器的总缓存内存量。

注意: 核武器工作室 支持使用 Python 脚本实现渲染场。脚本必须位于特定目录中，取决于平台，如中所示 启动期间运行 Python 。如果没有脚本存在, 渲染背景任务 下拉列表中的 出口 对话框被禁用。

一些示例脚本包含在 核武器工作室 包在以下目录中:

• 在 Mac 上:/应用程序/ 核武器工作室 9.0v1 / 核武器工作室 9.0v1 。应用程序/内容/插件/站点包/hiero/示例/

• 在 Linux 上:/usr/local/ 核武器工作室 9.0v1 /Plugins/site-packages/hiero/examples/

• 在 Windows 上: 驱动器号: \ 程序文件 \ 铸造 \ Hiero 9.0v1 \
插件 \ 站点包 \ hiero \ 示例
