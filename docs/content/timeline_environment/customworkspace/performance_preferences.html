性能偏好

这些 首选项允许您自定义 Nuke 工作室 的性能根据您的机器的能力。请参阅 优化读取和解码性能 欲了解更多信息。

偏好

默认值

描述

每个读取器的默认线程数

2

设置每个读取器要使用的线程数。如果您的源文件位于高性能本地驱动器上，增加线程数量可以显著提高读取时间。

CPU 密集型操作，如 。Jpg 解码，也可以通过增加每个读取器的线程数来提高。

每个读取器覆盖的解码线程数

没有

允许您覆盖根据文件格式使用的默认解码线程数。

使用 加 按钮将条目添加到表中，然后使用下拉菜单选择文件格式。双击 线程数 列以设置该格式所需的解码线程数。

要使用的 OpenEXR 帮助线程

0

设置仅用于 OpenEXR 的帮助线程数。默认值零自动设置使用的帮助器线程数。

要使用的 ARRI 帮助线程

0

设置仅用于 ARRI 的帮助线程数。默认值零自动设置使用的帮助器线程数。

注意: OpenEXR 和 ARRI helper 线程首选项独立于每个读取器的线程和每个格式设置的覆盖表。

QuickTime 解码器

8 总数

设置可用于处理 QuickTime 文件 I/O 的后台进程数。要使此首选项更改生效，必须重新启动应用程序。

注意: 使用太多解码器会影响性能，具体取决于可用的硬件。

启用 PBO 纹理上传

禁用(disabled)

启用像素缓冲区对象 (PBOs) 可以提高硬件、软件和文件格式的某些组合的播放性能。

注意: 您必须重新启动 Nuke 工作室 此选项将生效。

将图像从每像素 3 到 4 个通道展开

依赖于操作系统

启用此选项可以提高某些硬件组合的播放性能，而牺牲缓存的帧数。

注意: 默认情况下，在 Mac 和 Windows 上启用扩展。

播放缓存大小

33%

设置可用作缓存的总内存百分比。

您不能将其设置为大于可用内存 80% 的值 (四舍五入到最接近的半 GB)。例如，如果您有 6 GB 可用内存，最大可用缓存大小为 4.5 GB。

将缓存锁定到内存中

(仅限 Mac)

已启用

启用后，将播放缓存锁定在内存中，以便操作系统不会交换分配给播放缓存的内存。

此设置可以提高播放性能，但会减少运行其他应用程序的可用内存量。

注意: 缓存锁定仅在 Mac OS X 上可用。

当应用程序在后台时解锁缓存

(仅限 Mac)

已启用

当 将缓存锁定到内存中 是活动的，启用此控制可以通过允许其他应用程序在后台运行时使用保留的系统 RAM 来提高系统性能。

请记住，当应用程序再次变为活动状态时，可能会出现播放延迟，而缓存内存被重新交换。

注意: 缓存版本仅在 Mac OS X 上可用。

当应用程序进入后台时暂停缓存

已启用

启用后，当应用程序失去焦点时暂停 RAM 缓存。当你点击返回 Nuke 工作室 ,缓存从它停止的地方开始。

当应用程序进入后台时清除缓存

禁用(disabled)

启用后，当应用程序失去焦点时，会刷新 RAM 缓存。当你点击返回 Nuke 工作室 ,缓存再次从播放头的位置开始。

缩小过滤

1x

设置平均采样数，以实现列出位深度的代理分辨率。请参阅 使用缩减采样 欲了解更多信息。
