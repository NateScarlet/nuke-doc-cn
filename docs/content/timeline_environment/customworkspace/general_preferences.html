General Preferences

These preferences take care of how the applications start up, how often your work is automatically saved, localization, and cross-platform compatibility.

Preference

Default Value

Description

Startup Workspace

 Conforming

Sets the default startup workspace.

Show splash screen at startup

 enabled

Enables or disables the product page during startup.

Reconstruct directories on ingest

 enabled

When enabled, preserve the original folder/file hierarchy during ingest.

Show startup dialog

 enabled

Enables or disables the Settings dialog during startup.

Scan for file sequence range on drop into bin view

 enabled

When enabled, dragging a single file or file range, that is part of an image sequence, creates a clip in the bin view containing all available frames.

When disabled, only the dragged frame or range is imported into the bin.

See Using Drag-and-Drop for more information.

Automatically rescan versions when moving off end of the version list

 enabled

When enabled, incrementing a clip or track item’s version past the end of the previously discovered versions list, forces a rescan to update the versions list. See Using Versions for more information.

Warn when opening a locked Nuke Studio project

((Undefined variable: Custom.ProductPlayer) only)

  

When enabled, opening a Nuke Studio project in (Undefined variable: Custom.ProductPlayer) displays a read only warning.

If you disable this option, you won’t be warned but the clips and sequences in the project will still be colored light blue to remind you of the read only state of the project.

Autosave period

 5 minutes

Sets how often you want to save your work in the background. Use the up and down arrows or type a new value in the field.

Restore workspace when opening projects

 enabled

When enabled, restore the saved workspace layout at project load.

When disabled, use the Startup Workspace when you open an existing project.

Automatically save RMD files

 disabled

When enabled, automatically save changes to a RED clip’s Look effects in an RMD file.

Temp Directory

 Dependent on OS

Defines the directory used to cache thumbnail information.

You can disable caching using an environment variable, as described in Environment Variables.

Auto Localize From

 none

Sets the working directory that is automatically cached on ingest to the Localize To directory.

For example, setting the filepath to /project/myclips/footage would locally cache media from that directory on ingest, up to the Max localized files limit.

Localize To

 none

Sets the local directory where cached files are stored. See Caching Media Locally for more information.

NOTE:  If this filepath is not specified, cached files are saved in a subdirectory of the folder specified in Temp Directory.

Max localized files (MB)

 10000 MB

Sets the maximum space available for localized caching. Negative values in this field reserve the specified amount of space at all times. For example, -2000 stops 2 GB of memory being used for caching.

Path substitution

 none

Sets the mapping rules for cross-platform project sharing.

The mapping rules allow the application to read platform specific file paths by explicitly mapping them to Mac/Linux/Windows friendly names.
