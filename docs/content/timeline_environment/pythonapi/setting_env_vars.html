设置环境变量

设置环境变量的过程因您使用的操作系统而异:

在 Mac OS X 和 Linux 上

1。 第一步是确定您正在使用的外壳类型。启动终端并输入:

Echo $ SHELL

您的外壳类型将打印到屏幕上。例如, /Bin/bash 指示您的 shell 是 “bash”。

2. 根据 $ SHELL 命令的输出，执行以下操作之一:

• Csh 或 Tcsh -将以下命令添加到 。Cshrc 或 。Tcshrc 主目录中的文件:

Setenv 变量值

将变量替换为环境变量的名称，并将 value 替换为要赋予它的值，例如:

Setenv HIERO_PLUGIN_PATH/SharedDisk/ Nuke 工作室

• Bash 或 Ksh -将以下命令添加到 。Bashrc 或 。Kshrc 主目录中的文件:

导出变量 = 值

Replace VARIABLE with the name of the environment variable and value with the value you want to give it, for example:

导出 HIERO_PLUGIN_PATH =/SharedDisk/ Nuke 工作室

注意: 这些步骤将设置环境变量，直到您手动将其从主目录中的文件中删除。如果要为单个会话设置变量，只需在终端中键入变量，然后运行 Nuke 工作室 来自同一个会话。

在窗户上

1。 右击 on 我的电脑 并选择 属性 .
2. 单击 高级 标签。
3. 单击 环境变量 按钮。

的 环境变量 对话框打开。

4. 单击 新 按钮下 用户变量 或 系统变量 ,这取决于您是要为当前用户还是所有用户设置变量。

注意: 您必须具有管理员权限才能设置系统范围的变量。

5. 在 变量 名称字段中，输入要设置的环境变量的名称。
6. 在 价值 字段中，输入变量的值。例如，该值可以是目录路径。
7. 单击 好 保存变量。

下表列出了环境变量 Nuke 工作室 识别:

环境变量

描述

_ _ GL_SYNC_DISPLAY_DEVICE

在 Linux 上，将此变量设置为在双屏设置中同步的屏幕设备的名称。请参阅 在 Linux 上同步到 VBlank 欲了解更多信息。

FN_DISABLE_LICENSE_DIALOG

默认情况下，如果您安装了临时许可证, Nuke 工作室 在启动时显示一个对话框，提醒您剩余天数。如果要禁用此行为，可以将此环境变量设置为 1，以抑制关于即将到期的许可证的警告消息。

注意: 设置此环境变量后，如果未找到许可证，仍将显示警告消息。

HIERO_DISABLE_THUMBNAIL_CACHE

将此变量设置为 TRUE 以停止 Nuke 工作室 缓存缩略图。

注意: 此变量不清除缓存，必须手动删除缓存文件。

HIERO_PLUGIN_PATH

将此变量设置为启动时要运行的其他脚本的位置。你可以使用 。<文件夹> 指向隐藏的文件夹，例如 。MyPlugins .

例如，您可以创建一个设施范围的位置来共享预设，并使用此变量指向多个 Nuke 工作室 它的。请参阅 共享预设 欲了解更多信息。

使用以下方法显示设置路径:

打印 hiero.core.pluginPath ()

HIERO_SINGLE_THREADED_PLAYBACK

此变量启动 Nuke 工作室 在单线程模式下，可以解决各种 Linux Fedora 风格的回放问题。

FOUNDRY_LOG_FILE

此变量指定的位置 Nuke 工作室 的日志文件。如果不指定日志文件，所有输出都将显示在屏幕上。

FOUNDRY_LOG_LEVEL

此变量设置日志记录的级别 Nuke 工作室 在操作过程中产生。有四个细节级别，从最小到详细的滑动比例:

• 错误

• 警告

• 消息

• 详细

注意: 当指定 FOUNDRY_LOG_FILE 时，将日志记录级别设置为 verbose 可以生成大型日志文件。

OCIO

将此变量设置为用于颜色转换的 OCIO 配置文件的位置。

注意: 如果计划使用首选项中指定的 OCIO 配置文件，请确保 偏好 > 色彩管理 > 导出 Nuke 时使用 OCIO 节点 复选框已启用。
