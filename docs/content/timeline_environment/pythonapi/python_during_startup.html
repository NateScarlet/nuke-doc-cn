Running Python During Startup

On startup, Nuke Studio scans various directories for Python scripts that customize the behavior of the application. These scripts can be downloaded from online resources and placed in specific locations on disk, depending on the platform you’re using.

If you’re familiar with Python scripting, you can place custom startup scripts in the same locations, though bear in mind that you should take care as mistakes can produce performance issues.

Some example scripts are included in the application package in the following directories:

On Mac:

• /Applications/Nuke Studio9.0v1/Nuke Studio9.0v1.app/Contents/Plugins/site-packages/nuke/examples/

On Linux:

• /usr/local/Nuke Studio9.0v1/Plugins/site-packages/nuke/examples/

On Windows:

• drive letter:\Program Files\The Foundry\Nuke Studio9.0v1\plugins\site-packages\nuke\examples

At startup, Nuke Studio searchs <path>/Python/Startup and <path>/Python/StartupUI for any Python .py modules or packages containing __init__.py. Scanning is done in two passes - all the Startup folders are searched and then all the StartupUI folders.

You can also add your own Startup folders, which are scanned after the standard paths, by creating a Python file pointing to the custom location. For example:

import hiero.core

hiero.core.addPluginPath("/myCustomPlugins")

Searches /myCustomPlugins/Python/Startup after the standard paths.

Each package or module discovered is imported and added to the built-in package hiero.plugins.

NOTE:  Python startup scripts are imported in alphabetical order so you can force ordering if necessary.

The <path> varies by platform as follows:

Mac OS X

• /Applications/Nuke Studio9.0v1/Nuke Studio9.0v1.app/Contents/Plugins

• /Library/Application Support/TheFoundry/Nuke Studio

• ~/Library/Application Support/TheFoundry/Nuke Studio

• ~/.nuke

Linux

• /usr/local/Nuke Studio9.0v1/Plugins

• ~/.nuke

Windows

• drive letter:\Program Files\Nuke Studio\plugins\nukescripts

• drive letter:\Program Files (x86)\Nuke Studio\plugins\nukescripts
