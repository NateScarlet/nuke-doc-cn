Retiming Clips

In addition to transitions, Nuke Studio supports constant retimes on clip instances. Decreasing the speed of a clip instance causes frames to play more than once, whereas increasing the speed skips frames.

NOTE:  Audio is not currently supported for retimes and is automatically muted to avoid playback problems.

To retime clip instances using the Speed column in the spreadsheet:

1.   Select the required event(s) in the spreadsheet view.
2.   Click the cog icon and select the required Retime method:

• Keep source duration - the clip instance length is altered on the timeline depending on the retime applied.

For example, retiming a clip instance to 50% renders frames 1, 1, 2, 2, 3, 3, 4, 4, and so on in the Viewer, and as a result, the item’s length is doubled on the timeline.

Retiming a clip instance to 200% renders frames 1, 3, 5, 7, and so on in the Viewer, but the item’s length is halved on the timeline.

• Keep timeline duration - the clip instance length on the timeline is maintained regardless of the retime applied.

For example, retiming a clip instance to 50% renders frames 1, 1, 2, 2, 3, 3, 4, 4, and so on in the Viewer, but the item’s length on the timeline remains the same, effectively removing the second half of the item.

Retiming a clip instance to 200% renders frames 1, 3, 5, 7, and so on in the Viewer, but the item’s length on the timeline remains the same. If no extra frames are available from the source, the item is filled with black frames.

3.   Double-click the Speed column and enter the retime value.

To retime a clip instance using the Timeline menu:

1.   Select the required clip instance(s) on the timeline.
2.   Navigate to Timeline > Retime.

The Clip Speed dialog displays.

3.   Enter the required retime value as a percentage.
4.   Select the required retime method using the dropdown:

• Keep timeline duration - the clip instance length on the timeline is maintained regardless of the retime applied. When increasing speed, if no extra frames are available from the source, the clip instance is filled with black frames.

• Keep source duration - the clip instance length is altered on the timeline depending on the retime applied. For example, a 200% retime halves the length of the item.

• Anchor current frame - the clip instance length on the timeline and the current frame’s position are maintained after the retime. When increasing speed, if no extra frames are available from the source, the clip instance is filled with black frames.

5.   Click OK to retime the clip instance(s).
