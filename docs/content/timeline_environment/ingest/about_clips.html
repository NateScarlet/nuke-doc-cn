About Clips

The interface sorts your clips into three broad categories: Audio and Video, Audio Only, and Video Only. Clips are displayed differently depending on their content, location, and in the case of the Viewer, the current mode.

NOTE:  The timeline Viewer currently treats all alpha channels as premultiplied, which can result in the Viewer background being "added" to the image. If youâ€™re working with un-premultiplied images, set the Viewer background to Black.

NOTE:  There is currently no QuickTime audio support on Linux. Support for audio is scheduled for a later release.

Source Clips

Source clips are representations of files on disk. Clip instances on the timeline refer to source clips, so making changes to a clip in a bin affects all clip instances that refer to that source clip.

NOTE:  The colored bars under the thumbnail represent the layers available in the clip, in this case color. Other layers include alpha, depth, and motion, similar to Nuke.

Clip Instances

Clip instances are representations of source clips, they are not saved on disk. Making changes to a clip instance only affects that instance. See Managing Timelines for more information.

A Clip Opened in the Timeline View

Opening a clip in the timeline view allows you add in/out points and soft effects to the source clip. Clips opened as timelines always contain all available frames and can't be trimmed. See Using In and Out Markers and Soft Effects for more information.
