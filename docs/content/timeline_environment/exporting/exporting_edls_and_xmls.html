导出 EDLs 和 xml

Nuke 工作室 支持使用非常相似的方法导出到 EDL 和 XML，主要区别在于 EDL 不支持单个文件中的多个视频轨道，而 XML 支持。

注意: Nuke 工作室 可以读取 AAF 文件，但不能写出它们。

要导出为 EDL 或 XML:

1。 在 bin 视图中选择一个序列并导航到 文件 > 出口.

的 出口 对话框显示。

2. 选择 按顺序处理 从 出口 下拉列表。
3. 选择 CMX 3600 EDL 或 Final Cut Pro XML 预设，或复制一个并创建自己的预设。

注意: EDLs 每个文件只支持一个视频轨道。如果您有多个轨道，请包括 {轨道} 标记在快照模板中，为每个轨道预设写出一个 EDL。

例如, {Filename} _ {track}。{ext} 可能会为时间线上的每个轨道生成一个单独的 EDL，称为 MyTimeline_Video1.edl , MyTimeline_Video2.edl 等等。

4. 输入 导出到 目录或单击 选择…… 并浏览到该位置。

的 导出到 目录是快照模板构建快照层次结构的起点。

5. 如果要导出为 EDL，请在 内容 选项卡，如果需要:

• 卷轴名称 -定义写入 EDL 的卷轴名称，独立于剪辑的卷轴名称。

在此字段中输入文本或标准快照级别标记。请参阅 使用快照模板 欲了解更多信息。

如果该字段留空，则使用剪辑中的卷轴名称或剪辑实例的名称 (如果不存在卷轴名称)。

• 截断卷轴名称 -限制 卷轴 命名为八个字符。

• 使用绝对路径 -将每个剪辑的完整文件路径添加到 EDL 注释字段。

• 从剪辑名称 -定义 EDLs 中附加到 “从” 注释字段的文本，如 * 从剪辑名称 。文本和/或标准快照级别令牌在此字段中有效: {shot} 、 {clip} 、 {track} 、 {sequence} 、 {event} 、 {fps} 、和默认的 {filename}。

OR

如果要导出为 XML，可以启用 包括标记 将序列中出现的任何帧标记转换为 Final Cut Pro 或 Premiere 中的标记。请参阅 使用查看器标记 有关向框架添加标记的更多信息。

6. 单击 轨道和范围 选项卡并选择 此出口的轨迹 通过启用或禁用列表中的轨迹。 Nuke 工作室 出口 所有轨道 默认情况下。
7. 如果在序列上设置了输入和输出点，请启用 输入/输出点 仅导出选定的帧。
8。 设置如何剪辑 开始帧 使用下拉菜单导出:

• 序列 -使用序列的开始帧。

• 自定义 -使用右侧的字段为所有剪辑指定开始帧。

9. 设置 版本 出口编号 (如果适用)。使用箭头增加版本号，使用 +/-按钮增加或减少填充。也可以直接在数字字段中键入。

注意: 请参阅 使用版本 有关版本控制如何在中工作的更多信息 Nuke 工作室 .

10. 设置如何 Nuke 工作室 要使你的出口, 渲染与 下拉列表。 Nuke 工作室 提供以下选项:

• 帧服务器 -使用多个核弹进程来加快渲染时间 并与时间线中的任何 Nuke Comp 渲染共享渲染队列 ,改善资源管理。

请参阅 在外部机器上使用帧服务器 欲了解更多信息。

• 单一渲染过程 -使用一个单一的核弹过程来呈现你的出口。渲染 QuickTimes 回到此设置，但当帧服务器检测到问题时也会使用它。

• 自定义渲染过程 -使用自定义渲染过程。 Nuke 工作室 需要一个 Python 脚本将导出传递到您选择的渲染场。脚本必须位于特定目录中，取决于平台，如中所示 加载 Gizmos 、 NDK 插件以及 Python 和 Tcl 脚本 。如果不存在脚本，则下拉列表仅包含默认渲染进程。

11。 单击 出口 .

的 导出队列 窗口显示每个导出组件预计需要多长时间的估计。

导出完成后，将创建快照模板中指定的文件结构。

提示: 单击放大镜图标在浏览器窗口中显示导出的文件。
