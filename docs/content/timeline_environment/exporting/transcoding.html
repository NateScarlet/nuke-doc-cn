转码

转码 Nuke 工作室 使用背景渲染过程将一种文件格式转换为另一种文件格式。您可以从 bin 视图对序列、时间线选择和剪辑进行转码。

对序列进行转码

1。 在 bin 视图中选择一个序列并导航到 文件 > 出口.

的 出口 对话框显示。

2. 选择 按顺序处理 和您打算使用的预设，或使用默认 Dpx 预设。
3. 输入 导出到 目录或单击 选择…… 并浏览到该位置。
4. 单击 内容 在快照模板中显示转码选项的列。
5. 设置以下所有文件类型通用的控件:

• 渠道 -使用下拉列表设置要导出的通道。如果要导出非标准通道，请手动在字段中输入通道名称。

• 色彩空间 -使用下拉菜单设置要渲染的颜色空间，例如 线性 , REDLog ,或 Raw .

6. 选择 文件类型 使用下拉列表进行渲染并完成相关字段，具体取决于 文件类型 已选择。

注意: 选择 Mov 从下拉列表中提供了额外的 QuickTime 特定控件，允许您选择 编解码器 , 编码器 ,在某些情况下, YCbCr 矩阵 。矩阵控制使您能够使用新的 Rec 601 和 Rec 709 或的 遗产 编码方法，这是以前在 Nuke 中使用的方法。还有一个高级下拉列表包含 Mov32 和 Mov64 编码器特定的控制。

同样，选择 Exr 提供额外的元数据下拉列表，允许您导出或往返选定的元数据以及您的 。Exr 输出。

7. 使用 重新格式化 控制，以确定如何写点设在 Nuke 脚本:

• 没有 -使用剪辑或序列分辨率，在导出过程中不应用其他格式。

• 要缩放 -激活所有的 重新格式化 控件，除外 格式 ,允许您自定义导出分辨率。

• 自定义 -激活所有的 重新格式化 控件，除外 规模 ,允许您自定义导出分辨率。

重新格式化控制

要缩放

自定义

格式 -设置在 Nuke 中渲染的格式，如 1920x1080 高清 1080 .

规模 -设置缩放输出格式的比例。

调整大小 -设置要保留或覆盖原始纵横比的方法:

• 宽度 -缩放原始文件，直到其宽度与格式的宽度相匹配。然后以保持原始纵横比的方式缩放高度。

• 高度 -缩放原始文件，直到其高度与格式的高度相匹配。然后以保持原始纵横比的方式缩放宽度。

• 适合 -缩放原始图像，直到其最小边与格式的最小边相匹配。原始的较长的边然后以这样一种方式缩放，以保持原始的纵横比。

• 填充 -缩放原始图像，直到其最长边与格式的最长边相匹配。然后以保持原始纵横比的方式对输入的较短侧进行缩放。

• 扭曲 -缩放原始图像，直到其所有边与格式指定的长度匹配。此选项不保留原始纵横比，因此可能会发生失真。

中心 -启用后，将图像转换到输出的中心。禁用时，图像与输出的左下角对齐。

过滤器 -设置用于变换像素的过滤算法。请参阅 选择过滤算法 有关过滤器的更多信息 Nuke .

8。 完成所有文件类型通用的常规控制:

• 选择 重定时方法 申请 (如果适用)。

• 添加到导出中的剪辑实例的软效果包含在结果中 Nuke 默认脚本。如果你不需要软效果，禁用 包括效果 从脚本中省略它们。请参阅 软效果 欲了解更多信息。

• 包括音频 -启用后，任何音轨都将与视频一起导出。

• 删除。WAV 文件 -禁用时，删除 。Wav 用于创建导出的文件。此控件仅适用于支持音频的格式，如 。Mov 文件。

9. 启用 刻录小发明 使用 Nuke 小玩意刻录文本。单击 编辑 定义在老化过程中应用的信息。请参阅 将刻录文本添加到导出中 欲了解更多信息。
10. 指定任何 其他节点 在导出过程中需要单击 编辑 。请参阅 在导出期间添加其他节点 欲了解更多信息。
11。 检查 保持 Nuke 脚本 如果你需要 。Nk 转码操作后的文件。

注意: 以下控件可能会改善某些导出的渲染时间:

• 对于 。Dpx 导出，您可以启用 阅读所有行 ,这可以加快 I/O 重脚本的转码时间。

• 对于使用多个 CPU 插座的机器 单一渲染过程 ,你可能会发现限制渲染 使用单插座 可能会改善渲染时间。

轨迹和范围设置

1。 设置 版本 出口编号 (如果适用)。使用箭头增加版本号，使用 +/-按钮增加或减少填充。也可以直接在数字字段中键入。

注意: 请参阅 使用版本 有关版本控制如何在中工作的更多信息 Nuke 工作室 .

2. 选择 此出口的轨迹 通过启用或禁用列表中的轨迹。 Nuke 工作室 出口 所有轨道 默认情况下。
3. 如果在序列上设置 “入点” 和 “出点”，则启用 “入点/出点” 仅导出选定的帧。
4. 设置如何剪辑 开始帧 使用下拉菜单导出:

• 序列 -使用序列的开始帧。

• 自定义 -使用右侧的字段为所有剪辑指定开始帧。

5. 设置如何 Nuke 工作室 要使你的出口, 渲染与 下拉列表。以下选项可用:

• 帧服务器 -使用多个核弹进程来加快渲染时间 并与时间线中的任何 Nuke Comp 渲染共享渲染队列 ,改善资源管理。

请参阅 在外部机器上使用帧服务器 欲了解更多信息。

• 单一渲染过程 -使用一个单一的核弹过程来呈现你的出口。渲染 QuickTimes 回到此设置，但当帧服务器检测到问题时也会使用它。

• 自定义渲染过程 -使用自定义渲染过程。 Nuke 工作室 需要一个 Python 脚本将导出传递到您选择的渲染场。脚本必须位于特定目录中，取决于平台，如中所示 加载 Gizmos 、 NDK 插件以及 Python 和 Tcl 脚本 。如果不存在脚本，则下拉列表仅包含默认渲染进程。

6. 单击 出口 .

的 导出队列 窗口显示每个导出组件预计需要多长时间的估计。

导出完成后，将创建快照模板中指定的包含转码文件的文件结构。

提示: 单击放大镜图标在浏览器窗口中显示导出的文件。
