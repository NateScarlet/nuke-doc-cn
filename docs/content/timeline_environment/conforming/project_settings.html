时间轴环境 项目设置

开始工作的好地方是定义默认值 项目设置 在导入序列之前，特别是在 edl 的情况下，因为它们可能不包含帧速率信息。 项目设置 仅适用于当前项目并覆盖 偏好 设置。

注意: 您可以修改 项目设置 后来，例如，当你摄入媒体。

要定义项目设置:

1。 导航到 项目 > 编辑设置 .

的 项目设置 对话框显示。

2. 单击 一般 用于设置项目的子菜单 名称 .
3. 单击 序列 设置默认的子菜单 输出分辨率 , 帧速率 ,和 开始时间码 用于当前项目中的新时间线，并在将新剪辑添加到时间线时设置剪辑格式。
4. 单击 视图 用于设置多视图或立体项目的子菜单。请参阅 立体项目 欲了解更多信息。
5. 单击 色彩管理 用于管理此项目的显示和文件颜色空间的子菜单。

请参阅 颜色管理设置 欲了解更多信息。

提示: 使用中的选项 偏好 > 项目默认值 > 色彩管理 将它们应用于所有新项目。

6. 单击 红色设置 子菜单来定义 默认视频解码模式 对于当前项目中的新 R3D 文件。此设置将覆盖 偏好 > 行为 > 文件处理 > 默认红色剪辑视频解码模式 现有项目的控制。请参阅 附录 A: 偏好 欲了解更多信息。

注意: 更改此设置不会更改已导入的 R3D 媒体的解码设置。

下拉列表包含一个滑动分辨率比例 FullPremium 到 十六岁 ,但请记住，更高的分辨率解码较慢。

7. 最后，单击 出口/往返 子菜单以选择默认 拍摄预设 使用时 您选择 创建 Comp 从时间线。请参阅 创建 Comp 欲了解更多信息 .

颜色管理设置

Nuke 工作室 使用 OpenColorIO 进行颜色管理。中的所有颜色空间 Nuke 工作室 ,无论是应用程序附带的颜色空间还是自定义的颜色空间都是在 OCIO 配置文件中定义的。

根据您正在使用的 OCIO 配置文件，可以在中设置许多颜色空间选项和角色 (颜色空间的别名) Nuke 工作室 。还有默认选项，这些选项会根据您使用的文件类型而改变。当选择默认选项时，颜色空间 Nuke 工作室 它的设置列在括号中。

TIP: Use the options in Preferences > Project Defaults > Color Management to apply them to all new projects.

1。 设置要为此项目使用的 OpenColorIO 配置。

Nuke 工作室 附带了许多默认配置，但您可以:

• 通过选择使用自定义 OCIO 配置文件 自定义 从 OpenColorIO 配置 下拉列表，然后输入文件路径，或

• 将您自己的配置添加到您的 。 Nuke 文件。请参阅 添加 OCIO 配置 欲了解更多信息。

更改配置将更新 默认颜色转换 因此。如果所选配置对某些转换无效，将显示警告。例如，如果您选择已发货 Iff 配置, 8 位 和 16 位 转换不兼容。

在这种情况下，不兼容的转换设置为 Raw 色彩空间。

2. 的 工作空间 Transform 决定了在导入和导出过程中应该转换什么颜色空间文件-它是由 的 时间轴环境 引擎盖下。

注意: 在早期版本中 Nuke 工作室 ,这个色彩空间是隐藏的，因为 线性 总是被选为 工作空间 。你可能会发现一些操作在色彩空间中工作得更好 线性 。例如，一些转换在 堵塞 色彩空间。

3. 你可以使用 默认颜色转换 下拉菜单以覆盖查看器、缩略图等中的剪辑如何转换为 工作空间 .
4. 的 Nuke 脚本项目设置 下拉决定是否 Nuke 工作室 使用从指定的配置中读取的 LUTs 或 Nuke 出口时的本地 LUTs。选择 OCIO 使相关的 OCIO LUTs 可用于每个项目的脚本中的读写节点。

添加 OCIO 配置

您可以将自己的 OCIO 配置添加到 Nuke 工作室 当它们变得可用时，例如 ACES 的新版本。您还可以添加旧配置以实现向后兼容性。

1。 导航到您的位置 。 Nuke 平台显示的文件。您可能需要创建一个 。 Nuke 文件夹 (如果不存在)。

• Linux:/users/登录名/。nuke

• Mac OS X:/Users/登录名/。nuke

• 窗户: ~ \.nuke

注意: 在窗户上, 。 Nuke 文件夹可以在 HOME 环境变量指向的目录下找到。如果没有设置这个变量 (这是常见的), 。 Nuke 目录位于 USERPROFILE 环境变量指定的文件夹下，该变量通常是 驱动器号: \ 文档和设置 \ 登录名 \ 或 驱动器号: \ Users \ 登录名 \

要找出是否设置了 HOME 和 USERPROFILE 环境变量以及它们指向的位置，请输入 % HOME % 或 % USERPROFILE % 进入 Windows 资源管理器的地址栏。如果设置了环境变量，它指向的文件夹将被打开。

2. 在您的数据库中重新创建以下结构 。 Nuke 文件夹:

~/Plugins/OCIOConfigs/configs/<配置名称>

3. 将配置的内容复制到 配置名称 命名文件夹。应该有一个 Luts 文件夹和 。Ocio 最低限度的文件。
4. 如果 Nuke 工作室 已在运行，重新启动应用程序以应用更改。
5. 您现在可以从 项目设置 > 色彩管理 > OpenColorIO 配置 下拉列表。
