使用浏览器符合

使用浏览器符合序列:

1。 导入序列后，单击 匹配媒体 在电子表格上，使用浏览器找到包含正确媒体的源文件夹。

注意: 匹配媒体 也可以用于电子表格视图中的选定事件。

2. 单击 打开 要显示 符合选项 对话框。

Nuke 工作室 使用一组符合 规则 和文件名 模式 要按顺序将磁盘上的候选媒体文件与事件或剪辑实例匹配，请执行以下操作:

• 规则 -将脱机媒体属性设置为在符合期间与相应的电子表格条目匹配。

依赖事件或候选片段中不存在的信息的规则将被忽略，一些规则将其他规则复合，以确定更好的匹配。

规则

描述

Umid

将文件的唯一材质 ID (UMID) (创建时写入文件元数据) 与候选媒体的 UMID 匹配。如果其中一个或两个都缺少 UMID，则忽略此规则。

RedTapeName

将事件中的红色相机卷轴名称与候选媒体名称匹配。

RedName

在与候选媒体名称匹配的事件中查找红色样式的相机文件名。

ReelName

在候选人的媒体名称中查找事件的卷轴名称。

全路径

将事件的整个文件路径与候选媒体的整个文件路径匹配。

文件名

仅将事件的文件名 (无路径) 与候选媒体的文件名匹配。

文件头

将事件的文件名头 (无路径、文件扩展名或填充) 与候选媒体的文件名匹配。

PartialName

在候选媒体的名称中查找事件的名称，反之亦然。

FolderName

在候选媒体的文件路径中查找事件的名称。

默认情况下，所有规则都是启用的，但是如果规则导致特定编辑和源剪辑集之间的不正确匹配，您可能偶尔需要禁用这些规则。

提示: 使用 选择 / 取消选择全部 用于快速启用或禁用规则的按钮。

• 模式 -在符合期间设置包含和排除参数，以空格分隔。例如, * Mov * dpx 只会包括或排除 。Mov 和 。Dpx 文件。

您还可以按名称进行匹配，例如 BR_Shot * ,这将只包括或排除以该字符串开头的源剪辑名称。

提示: 在符合期间尽可能具体地搜索位置总是一个好主意，但是如果需要，符合 规则 和 模式 可以节省时间。

3. 启用 接受最佳时间码匹配. 如果没有匹配规则，则使用最近的源时间码匹配来符合事件。
4. 当 忽略具有非重叠时间码的剪辑 如果启用，则忽略任何可能匹配的源片段，其时间码根本不会与相关片段实例重叠。
5. 检查 符合已经有媒体的镜头 如果要更新所有时间线剪辑实例。默认情况下，应用程序不会尝试符合未脱机的事件。
6. 当 分裂序列 如果启用，则 conform 找到的任何非连续文件序列都将拆分为单独的片段，其方式与 分裂 seq 选项在文件浏览器中启用。
7. 单击 好 开始整合过程。

Nuke 工作室 尝试将编辑与选定的媒体相一致。

一致性完成后，将有一个对话框通知您成功率。

成功匹配的媒体被放置在新的 符合 项目中的 bin。

注意: 通过将光标悬停在所需条目上，可以显示为每个电子表格对象匹配的一致规则。
