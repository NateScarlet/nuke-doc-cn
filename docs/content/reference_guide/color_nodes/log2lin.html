Log2Lin

每当你将剪辑读入脚本时，它会自动转换为 Nuke 的本地颜色空间，即每通道 32 位 RGB，这是一种线性格式。即使您读取的剪辑是柯达 Cineon 格式，这是对数格式，也会进行这种转换。当您将处理的元素从脚本中重新写入时，这种转换的反向转换也会自动发生，称为 lin-to-log 转换，即, nuke 会自动将其转换回 Cineon 元素。

Nuke 在两个方向进行 Cineon 转换时使用柯达推荐的设置。很少你想重写这些设置，但是如果有必要，你可以使用 Log2Lin 节点。如果你这样做，你也应该检查 原始数据 (或设置 色彩空间 到 线性 ) 在读写节点控件中跳过自动转换。
