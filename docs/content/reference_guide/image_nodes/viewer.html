观众

“查看器” 节点显示中任何连接的进程节点的渲染输出 观众 面板。它不会以任何方式改变数据，但只允许您查看上游所做任何更改的效果。

您可以在脚本中放置任意多个查看器节点，这允许您同时查看多个输出。您还可以将多达十个进程节点的输出连接到单个查看器节点，然后循环显示各种显示。这允许您通过给定的效果轻松比较处理前后的图像。

您也可以通过以下方式创建此节点:

• 选择 图像 > 观众 从 Nuke 工具栏，或

• 按 Ctrl / Cmd 我 .

Gl 缓冲区深度

Gl_buffer_depth

浮动

设置 OpenGL 缓冲区深度，并允许将 GPU 用于查看器进程和输入进程:

• 字节 -转换为 8 位错误扩散。

• 半浮 -转换为 16 位 (半) 浮动。

• 浮动 -使用完整的 32 位浮点纹理 (在某些卡上可能非常慢)。

在半浮动模式和全浮动模式下，GPU 可用于在查看器过程中应用查看器效果，如 gamma 和 LUT。

注意: 你可以选择一个预设值的设定, 偏好 或通过使用 KnobDefault () 在启动脚本中。

尽可能为查看器使用 GPU

UseGPUForViewer

已启用

启用时，查看器效果 (如 增益 , 伽马 ,和查看器进程节点) 在可能的情况下使用 GPU 应用。

“可能时” 条款是因为在某些情况下，如启用广播监视器输出或 GL 缓冲区深度为 8 位时，必须在 CPU 中计算效果。

尽可能使用 GPU 进行输入

Usegpuforinput

已启用

当启用时，计算节点尽可能连接到 GPU 上的查看器输入，尽管不是所有节点都有 GPU 实现。

通常，观众只尝试运行自己的效果 (如 增益 , 伽马 ,和浏览器进程节点) 在 GPU 上。

注意: 如果在 GPU 上计算节点，查看器中显示的颜色值是不准确的，因为它们显示了 CPU 中计算的最后一个节点的颜色, 在将图像传输到显卡之前。
