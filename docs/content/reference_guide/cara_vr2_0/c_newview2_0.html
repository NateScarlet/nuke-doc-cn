C_NewView

使用 C_NewView 节点使用另一个视图的像素重建视图。例如，您可以选择使用右视图中的像素重建左视图。如果要操作一个视图 (例如，使用 gizmo 、节点或图形编辑器) 并将更改复制到另一个视图，这将非常有用。

注意: C_NewView 需要来自上游 c_dispararitygenerator 的视差向量来重建视图。

看到商队 在线帮助 有关 C_DisparityGenerator 的更多信息。

使用 GPU (如果可用)

UseGPUIfAvailable

已启用

启用时，渲染发生在 本地 GPU 指定 (如果可用)，而不是 CPU。GPU 和 CPU 之间的输出在 NVIDIA GPU 上是相同的，但是使用 GPU 可以显著提高处理性能。

Nuke 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上支持 AMD gpu，运行 OS X 10.9.3 “mavericks” 或更高版本。虽然，在某些情况下，GPU 和 CPU 之间的比特级相等保持不变，但对于某些操作，这种配置可能存在精度限制。

注意: 在没有本地 GPU 的情况下启用此选项允许脚本在 GPU 上运行，只要脚本在具有 GPU 可用的机器上打开。
你应该选择这个，如果你想使从命令行的 -- Gpu 选项。

GPU 处理还需要某些 NVIDIA GPU 和 CUDA 驱动程序，或者在 2013年底运行 10.9.3 “mavericks” 或更高版本的 Mac Pro 6,1 和 2015年中 Mac Pro 11,5 的情况下，某些 AMD GPU 和驱动程序。有关最低要求的更多信息，请参考 帮助 > 发行说明 .

修正

遮挡校正

使用原始

确定如何处理遮挡区域:

• 使用原始 -将原始视图保留在被遮挡的区域中。例如，如果使用原始左视图构建新的右视图，则原始右视图将保留在被遮挡的区域中，以帮助构建新的右视图。

• 展开前景 -通过从原始源视图扩展周围区域来填充被遮挡的区域。例如，如果您使用原始左视图构建新的右视图，则原始左视图的前景将展开以帮助构建新的右视图。

注意: 展开前景可能会偏移边缘。

• 使用清洁板 -使用干净的背景填充被遮挡的区域，使用 清洁板 输入。

• 没有 -不要填充被遮挡的区域。

修正

EdgeCorrection

匹配原始

确定如何处理深度边界上的图像边缘:

• 匹配原始 -匹配边缘原始视图的外观。例如，如果使用原始左视图构建新的右视图，则边缘与原始右视图匹配，以帮助构建新的右视图。

• 匹配前景 -匹配用于构建新视图的原始源视图的边缘。例如，如果使用原始左视图构建新的右视图，则边缘与原始左视图匹配，以帮助构建新的右视图。

• 没有 -不要应用边缘校正。
