合成的一个重要任务是去除蓝屏或绿屏。Nuke 为这项任务提供了各种各样的键控。我将非常简短地介绍其中的几个，这样你就可以介绍了。我带了两段录像来试试这个。有一个我们以前使用过的人的绿色屏幕，还有一个房子的静止图像。它只是有一个明亮的天空，即使在这张特定的照片中没有绿色或蓝色，你仍然可以使用 keyer 工具攻击天空并移除它。你可以到键工具, Keyer 菜单。

我将从最简单的一个开始，叫做 Keyer。现在，尽管它没有很多选择，但对于像这样的某些情况来说，这是非常好的。我要降下来了 一个 合并和读取节点之间的管道。Keyer 有几个操作，其中之一是 亮度键 ,这是默认的。您还可以将某些颜色 (如绿色) 或某些属性 (如饱和度) 作为目标。我们暂时让它保持亮度。这里还有一个图，它代表操作值。例如，此图表表示亮度，因为它从 0 到 1 或最大值。这里还有四个黄色条: 一个 , B , C ,和 D 。您可以通过交互方式单击并拖动这些对象。现在 B 和 C 一开始是重叠的，但是你可以把它们分开。一旦我移动这些，阿尔法哑光就会开始形成。让我们去看看。我将进入观众并按下 一个 关键; 这是阿尔法频道。这意味着任何亮度值在 B 和 C 之间的像素都变得不透明。任何亮度在 C 和 D 之间或 B 和 a 之间的像素值在不透明和透明之间有一个逐渐变细的值。所以，在这种情况下，我能做的是将 B 和 A 移动到最左边，然后调整 C 和 D，使天空变得透明。例如，这意味着亮度值超过 0.9 的任何像素都将变成 100% 透明，或者亮度值介于 0.8 和 0.9 之间的像素具有逐渐变细的透明度。

让我们回到 rgb。按下 一个 钥匙又来了，我们走了。现在，它看起来不像是正在发生的任何事情。你要做的是预乘阿尔法。所以，我将选择 Keyer 节点，点击鼠标右键, 合并 > Premult 。Premult 将 alpha 值乘以 rgb 值。一旦我添加了这个，天空就被移除了。现在我可以进一步测试了。我可以把东西钩到 B 合并的管道。例如，我可以用鼠标右键单击，然后转到 图像 > 常数 。常数会产生纯色 -- 我会把它钩到 B 管道。然后转到该节点的色轮，选择一种颜色，如浅蓝色。天空消失了，现在常数出现在天空区域。

Let’s move on to some other keyers. I am going to go back to the greenscreen footage. Now, in fact, you can have more than one Viewer in any project. You can make a new Viewer at any time by going to the 观众 > 创建新查看器 ( Ctrl / Cmd + I ). If you have more than one Viewer, you can switch between them by clicking the tabs. So now, we are going to work on this greenscreen. The first keyer we will try here is Keylight, so Keyer > 键光 . Foundry writes Keylight and, in fact, it is available in other compositing packages like After Effects, the functionality is the same. It looks a little intimidating at the start because of all the inputs. What you have to do is plug the 源 into the greenscreen, in this case, the A pipe into the output. Now there are many options you can adjust. I am going to adjust the basic ones you need to remove the greenscreen. The first thing to do is select the 屏幕颜色 . You can click the swatch here, get the eyedropper, and go back to the Viewer. There are several ways to sample pixels with the eyedropper. You can Ctrl / Cmd +click or Ctrl / Cmd +drag your mouse. I will try the drag, so click+drag, let go - the screen color is sampled and you will see it right here in the swatch, and also beside the Screen Colour property. Let’s take a look at the alpha. Now, Keylight offers this 视图 menu, which you can change from 最终结果 to 组合磨砂 . That’s what the alpha channel looks like, so right now there is some gray in the transparent area. What you can do is raise the 剪辑黑色 to erode that. What the Clip Black does is it looks for any pixel less than the slider and it makes it 100% transparent. There is also some gray in the white area. You can lower 夹白色 to make them more opaque, and now the matte looks pretty good. Once the matte looks good, you can return the View to Final Result . Now, to test that, I will disconnect the Constant over here, and plug it into the B pipe here. There we go, that’s pretty successful. Let’s try another keyer. I am going to disconnect Keylight, disconnect constant, and move that aside.

现在我们来试试灵长类动物。Primatte 已经更新了 Nuke 7，所以现在它更强大了。在这种情况下，我需要钩 Fg 或前景到绿色屏幕，然后输出到 一个 管道。Primatte 有一个非常强大的按钮叫做 自动计算 。如果你点击它，它会尝试识别屏幕颜色并删除它。事实上，它确实做得很好。让我们看看阿尔法是什么样子。我会按 一个 再次关键，它就在那里。同样，边缘也有一些噪音。幸运的是，Primatte 提供了一长串可以用来清理哑光的操作。例如，我可以将此菜单切换到 清洁 BG 噪音 。现在我会放大，现在，我可以采样那些太灰色的像素。另一种取样方法是 Ctrl / Cmd 转变 在问题区域周围画一个选框，同时对一大堆像素进行采样。

所以现在边缘看起来更好，但是他的夹克上仍然有一些灰色。然后我们可以将菜单切换到 清洁 FG 噪音 然后对这些像素进行采样。现在变得更加不透明。让我们再看看 rgb。按下 一个 关键是它在那里。它看起来很干净，但是，他衣服的绿色屏幕上有很多绿色溢出。那样的话，我可以去 溢出海绵 对这些像素进行采样，将绿色拉出。就这样。现在它还不完美，但是你可以看到你能多快移动绿色。让我们继续插入常量，看看它是什么样子。好吧，让我们再继续一个 keyer。我会断开这个，把它移到一边。

现在我们将尝试 IBKGizmo 和 IBKColor: 两个节点，它们被设计用来一起处理绿色屏幕或蓝屏。所以 Keyer > IBKColour 和 Keyer > IBKGizmo 。现在，这里的关系有点复杂。基本上, 1 IBKColor 的输入进入绿色屏幕，就像 Fg 对于 IBKGizmo。然后是 c 或 IBKGizmo 的颜色输入管道，转到 IBKColor 节点。然后是 一个 合并的管道将转到 IBKGizmo 的输出。让我们看看选项。首先要改变的是 屏幕类型 对于 IBKColor 节点。它被设置为 蓝色 默认情况下，但您可以将其更改为 绿色 。这在浏览器中看起来有点有趣，但是我可以做的是将浏览器插入该节点。在这里，它的目标是绿色; 无论什么颜色不是绿色，它都在移除。现在清除的侵略性是由 尺寸 滑块。如果我增加这一点，会有更多的平均，有越来越少的非绿色。让我们将观众重新插入 IBKGizmo。颜色信息从 IBKColor 节点传递到 IBKGizmo 节点，IBKGizmo 节点将其变成哑光。现在，你需要对 IBKGizmo 做的第一件事是确保 屏幕类型 设置为相同的颜色，在这种情况下, 绿色 。让我们将常数插入 B 管道并将查看器放回合并节点，然后我们就开始了 -- 成功的绿屏移除。
