C_SphericalTransform

C_SphericalTransform 在不同投影之间转换图像，类似于 vanilla Nuke SphericalTransform 节点，但设计用于 360 工作，包括闪烁 GPU 加速。这些视图预测可以分为两大类:

• 全帧，如 Latlong ,围绕一个点，包括整个 360 世界,

• 部分帧，如 直线 看来，核弹是用来工作的。

的 输出 旋转也可以通过浏览器控制系统进行控制。按住 Ctrl / Cmd Alt 然后左键单击并拖动以左右移动图像，设置平移和倾斜设置。添加 转变 锁定运动的单一维度。在部分帧投影中，使用鼠标右键设置 焦距 ,本质上放大和缩小。

看到商队 在线帮助 有关 C_SphericalTransform 的更多信息。

使用 GPU (如果可用)

UseGPUIfAvailable

已启用

启用时，渲染发生在 本地 GPU 指定 (如果可用)，而不是 CPU。GPU 和 CPU 之间的输出在 NVIDIA GPU 上是相同的，但是使用 GPU 可以显著提高处理性能。

Nuke 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上支持 AMD gpu，运行 OS X 10.9.3 “mavericks” 或更高版本。虽然，在某些情况下，GPU 和 CPU 之间的比特级相等保持不变，但对于某些操作，这种配置可能存在精度限制。

注意: 在没有本地 GPU 的情况下启用此选项允许脚本在 GPU 上运行，只要脚本在具有 GPU 可用的机器上打开。
你应该选择这个，如果你想使从命令行的 -- Gpu 选项。

GPU 处理还需要某些 NVIDIA GPU 和 CUDA 驱动程序，或者在 2013年底运行 10.9.3 “mavericks” 或更高版本的 Mac Pro 6,1 和 2015年中 Mac Pro 11,5 的情况下，某些 AMD GPU 和驱动程序。有关最低要求的更多信息，请参考 帮助 > 发行说明 .

元数据

MetaDataOperation

忽略

设置传入元数据的处理方式:

• 忽略 -流中存在的任何元数据都不会被更改通过下游。

• 输入 -根据流中的元数据和 C_SphericalTransform 节点中的控件描述的总变换旋转图像。

注意: 此选项假定传入图像在 LatLong 空间中。

• 输入输出 -将输出元数据设置为输入元数据和 C_SphericalTransform 节点中的控件描述的总变换的总和，但不修改输出图像。

过滤器

ResampleType

立方

选择将像素从原始位置重新映射到新位置时要使用的过滤算法。

作为一般准则，当您向下移动列表时，过滤器会以处理时间为代价提高质量。 双线性 过滤是最快的 Lanczos 最锋利的。

格式

格式化模式

要缩放

从输入开始设置要使用的格式或如何计算格式:

• 格式化 -强制指定的输入格式作为输出格式。当使用相机类型投影时，例如 直线 ,该 传感器尺寸 y 参数被忽略。

• 至宽度 -输出格式具有指定的输入宽度，但高度是根据输出计算的 投影 格式比率。

• 要缩放 -输出格式宽度根据输入格式宽度计算，高度根据输出计算 投影 格式比率。

C_SphericalTransform (反向链接)

设置要添加到节点图中的导出节点:

• C_SphericalTransform (反向链接) -创建链接 C_SphericalTransform 集的表达式，以反转当前转换。

当在直线和 latlong 之间切换以执行 VFX 任务 (如旋转) 时，这可能很有用。

• C_SphericalTransform (串联) -将当前 C_SphericalTransform 节点设置为 输入输出 模式并在中创建第二个 C_SphericalTransform 节点 输入 模式。

这可以用于创建增量 latlong 旋转，而不会引入不必要的过滤命中，方法是只将元数据传递到下游，直到节点树中的最终 C_SphericalTansform 执行级联旋转。
