C_Stitcher

C_Stitcher 使用 Ocula 风格的视差向量来排列重叠相机之间重叠区域中的匹配特征，然后混合结果以生成球形 latlong。

注意: C_Stitcher 需要上游的 C_CameraSolver 来生成图像。

使用 GPU (如果可用)

UseGPUIfAvailable

已启用

启用时，渲染发生在 本地 GPU 指定 (如果可用)，而不是 CPU。GPU 和 CPU 之间的输出在 NVIDIA GPU 上是相同的，但是使用 GPU 可以显著提高处理性能。

核武器 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上支持 AMD gpu，运行 OS X 10.9.3 “mavericks” 或更高版本。虽然，在某些情况下，GPU 和 CPU 之间的比特级相等保持不变，但对于某些操作，这种配置可能存在精度限制。

注意: 在没有本地 GPU 的情况下启用此选项允许脚本在 GPU 上运行，只要脚本在具有 GPU 可用的机器上打开。
你应该选择这个，如果你想使从命令行的 -- Gpu 选项。

GPU 处理还需要某些 NVIDIA GPU 和 CUDA 驱动程序，或者在 2013年底运行 10.9.3 “mavericks” 或更高版本的 Mac Pro 6,1 和 2015年中 Mac Pro 11,5 的情况下，某些 AMD GPU 和驱动程序。有关最低要求的更多信息，请参考 帮助 > 发行说明 .

阿尔法

在默认设置之间切换 阿尔法 模式和 多频段 模式:

导出菜单

ExportMenu

STMaps (分割)

设置要添加到节点图中的导出节点:

• STMaps (分割) -为缝合中的所有视图创建包含 stitch_map 通道的单个 STMap。

• STMaps (独立) -为缝合中的每个视图创建一个单独的 STMap，其中包含一个 stitch_map 通道。

• 手动 STMap 缝合 (分割) -创建一个工作流，通过包含一个 STMap 来扭曲原始输入 Stitch_map 缝合中所有视图的通道。

• 手动 STMap 缝合 (独立) -创建一个工作流，通过单独的 STMaps 扭曲原始输入，其中包含 Stitch_map 通道，用于缝合中的所有视图。

• PPass (分割) -创建一个包含 Ppass_map 缝合中所有视图的通道。

• PPass (独立) -创建一个单独的 C_STMap 节点，其中包含 Ppass_map 通道，用于缝合中的每个视图。

• 手动 ptass 缝合 (分割) -创建一个工作流，通过包含一个 C_STMap 扭曲来自 C_CameraSolver 的原始输入 Ppass_map 缝合中所有视图的通道。

• 手动 ptass 缝合 (独立) -创建一个工作流，通过单独的 C_STMaps 扭曲来自 C_CameraSolver 的原始输入，其中包含 Ppass_map 通道，用于缝合中的所有视图。

Enable Stereo Stitch

enableStereo

disabled

When enabled, C_Stitcher creates default left and right views and then uses on omni-directional rig model to generate a stereo stitch automatically.

NOTE: This automatic stitch method requires a greater amount of overlap between adjacent cameras than regular stitching.

When disabled, C_Stitcher generates a monoscopic stitch using all cameras.

眼罩分离

0.065

从观众的角度确定两个视图的距离。如果你设置 眼睛分离 ,或者瞳孔间距 (IPD)，太低，场景中的对象显示为水平压碎，但将其抬高得太高会在针脚上留下洞。

衰减类型

EyeSeparationFalloffType

余弦

确定如何处理极点合并:

• 没有 -没有 IPD 调整发生在两极。

• 线性 -视图从逐渐合并 开始角度 向极点指定的。增加角度会将起点向极点移动。

• 余弦 -视图向两极平滑合并。减少了 分离衰减 将深度转换转移到极点。

分离衰减

分离衰减

1

设置眼睛向两极分离时的衰减率 衰减类型 设置为 余弦 .

值为 1 会产生向左视图和右视图的极点的平滑合并。

向量选项卡

Enable Warps

enableWarps

enabled

When enabled, the linked cameras displayed on the Cameras tab are warped using the vectors generated at the keyframes specified.

When disabled, the input is not warped.

矢量细节

VectorDetail

0.4

控制计算向量的密度。较高的值可以获得较小的差异变化，但需要更长的时间来计算。

强度

强度

1

设置匹配相机之间匹配像素时应用的容差。

更高的值允许您准确地将一幅图像中的相似像素匹配到另一幅图像，即使结果向量是锯齿状的，您也可以专注于细节匹配。

较低的值可能会错过本地细节，但不太可能为您提供奇怪的伪向量，从而产生更平滑的结果。

一致性

一致性

1

设置匹配相机之间的相同点相互映射的精确程度。

增加该值以鼓励向量匹配。

迭代

MaxIteration

8

控制根据上一个迭代生成向量的时间。

增加此控制会产生更平滑的向量，但会增加处理时间。

扭曲

扭曲

4

控制生成过程中执行的像素比较范围。

增加此控制会产生更平滑的向量，但会增加处理时间。

时间窗口

临时窗口

0

将帧数设置为关键帧的任一侧，以执行时间平均。

增加这种控制会产生更平滑的向量，但会显著增加处理时间。
