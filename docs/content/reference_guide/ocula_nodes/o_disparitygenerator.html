O_DisparityGenerator

O_DisparityGenerator 插件用于为立体图像创建视差字段。一个 视差场 将一个视图中像素的位置映射到另一个视图中相应像素的位置。它包括两套 视差向量: 一个将左视图映射到右，另一个将右视图映射到左。

以下是 Nuke 插件依靠视差字段来产生输出:

• O_OcclusionDetector

• O_ColourMatcher (在 3D LUT 和 本地匹配 模式)

• O_FocusMatcher

• O_verticalalaligner (在 局部对齐 模式)

• O_NewView

• O_InteraxialShifter

• O_VectorGenerator

• O_DisparityToDepth，和

• O_DisparityViewer。

Ocula 中的 verticalalaligner 节点可以根据视差的垂直分量对输入镜头进行每像素更新，以水平对齐。我们可以看到这个，如果我们切换到浮雕这个镜头。如果我看这里的镜头，并将其与更新的垂直对齐镜头进行比较，它会进行每像素更新。

现在，您还可以使用 Ocula 中的 NewView 节点，使用 “视差” 从另一个视图重建一个视图。在这里，我们使用左边的图像通过使用视差向量将左边的像素穿过来重建右边。所以，与原始的正确视图相比，这是重建的正确视图。您也可以使用差异来推动 rotos。在这里，我在左眼创建了一个旋转装置，并将其推到右眼。最后，我们使用 Ocula 中的视差来创建左右眼一致的运动矢量。所以，视差将左右运动联系起来，确保我们对它进行的任何重新计时都是一致的。差异对所有 Ocula 行动至关重要。让我们看看现在设置 DisparityGenerator，并回顾这些差异向量。

在这个例子中，我在前景舞者周围画了一个粗糙的旋转。现在，如果我们看看原始图像的差异，忽略生成器 (棋盘) 节点，你可以看到舞者已经被有效地画了出来，给了我们一个完美的平滑, 可用的背景视差板。如果我们将这种差异提供给 NewView 节点，你可以看到它做得更好。眼睛之间的推动后，直线保持直线。如果我们重新组合舞者，你可以看到棋盘不再受到舞者的影响，并且随着时间的推移保持光滑。现在让我们在原始背景板重建和我们的层增强重建之间进行交换，以更清楚地看到效果。这个例子展示了使用 Ocula 的视差层有多强大，更不用说在不需要相机跟踪和几何投影的情况下在视图之间推动背景变化有多容易了。

在本教程的下一部分，我们将开发前面展示的技术，以便为背景和前景区域获得更好的新视图。在这种情况下，想象一下你一直在努力修复和治疗舞者和左眼的背景元素, 以至于右眼不再与左眼有任何相似之处。现在，您希望在其他视图中推送这些编辑。虽然相机跟踪和投影工作流将适用于背景元素，但它可能不适用于前景。所以，很自然，我们转向 Ocula 的 NewView 工作流来帮助我们。首先，让我们看看最简单、最标准的方法，通过将 O_NewView 节点连接到 o_dispalitygenerator 来从视差中创建新视图。正如你所看到的，它没有做得很好。我们在舞者的边缘周围有很多失真和人工制品，背景的部分扭曲不正确，比如道路的中心和路缘。为了解决这个问题，我们将巧妙地利用视差生成器、忽略和前景蒙版分别从前景和背景中提取视差, 然后在最后结合结果，给出另一个视图更清晰的构建。在本教程的前面，我们看到了如何使用忽略遮罩提取背景差异，所以让我们现在开始使用它。让我们将与以前相同的粗糙旋转蒙版应用于 O_DisparityGenerator。看看 NewView 节点，我们现在可以看到背景处理得更好，沿着道路的拐角处和中心线有很好的直边。然而，因为我们决定忽略舞者的区域，我们基本上放弃了差距，所以我们现在必须分别估计。还要注意，在估计新视图之前，我们正在用舞者的阿尔法面具洗牌。这将在以后使用。

为了准确地提取舞者的差异，我们将向 O_DisparityGenerator 节点提供舞者的 alpha 掩模作为前景掩模。这明确地告诉视差生成器，它必须在两个视图中匹配遮罩中的区域。现在，当我们从前景视差中创建一个新的视图时，我们可以确定背景不会影响舞者的视差。例如，看着 NewView，你可以看到唯一几乎不扭曲的区域是舞者，这正是我们想要的。现在，让我们用舞者的 alpha 来组合前景和背景。除了舞者的重影效果，新盘子看起来非常漂亮。特别是，我们没有看到围绕舞者或背景的任何奇怪的扭曲。幽灵效应是由于这些区域在左视图中被遮挡，所以当我们从左视图构建右视图时, 我们没有正确的数据来重建。虽然我们的大多数 “英雄” 观点都被正确地转移了，但这些幽灵区域需要由一名 comp 艺术家来解决，可能会应用与 “英雄” 观点相同的处理方法。通过一些简单的掩模操作，您可以快速识别这些区域。

为了结束这个教程，我将欺骗一点，假装我画了这些缺失的部分，简单地从原始右眼复制过来。虽然颜色有点偏，因为我们还没有进行任何颜色匹配，但是你可以看到原始右眼边界与我们的新视图重建有多吻合。这让我们知道我们的板重建工作流程做得很好。你可以用我们网站上附带的脚本和镜头亲自尝试这些技术。这结束了这个关于使用视差图的 Ocula 教程。
