O_Retimer

O_Retimer 设计用于使用 O_VectorGenerator 节点生成的上游运动矢量来减慢或加快立体声镜头。这些运动矢量描述了每个像素如何从一帧移动到另一帧。有了精确的运动矢量，通过沿运动方向插值，可以在整个序列的任何时间点生成输出图像。

输入和控制

连接类型

连接名称

输入

运动

如果这里提供了立体图像和 O_VectorGenerator 节点，则将根据该序列计算运动矢量并应用于输入序列。

例如，如果你的输入序列非常嘈杂，因为太多的噪音会干扰运动估计，这可能会很有用。在这种情况下，您应该在这里提供序列的平滑版本和 O_VectorGenerator 节点。

源

一对立体图像。

如果运动矢量没有嵌入到图像中，您应该使用 O_VectorGenerator 节点来计算它们。

• 源帧 -如果你想用 “在输出剪辑的 100 帧，我想看到源剪辑的 50 帧” 来描述重新计时，请选择此选项。您需要为此设置至少两个关键帧，以重新计时剪辑。

• 正常 -此模式扭曲和混合图像，使其保持一致。

• 尖锐闭塞 -此模式旨在减少具有不同边界的 CG 源的人工制品。

好的，现在让我们看看重定时。我有我的校正板与视差和我刚刚计算的运动矢量。我把它们拉进一个 Ocula 3.0 重定时器节点。重定时器很简单。它允许你把 a 速度 控件，也可以指定帧。您可以定义提供给帧的扭曲，以产生重定时结果。所以让我们看看左视图和右视图重新计时的结果。你会注意到这里的手有一些快速运动，这在重定时结果上造成了一些重影。你可以看看使用更高的 强度 ,或者也许 锐度 ,在运动矢量中纠正这一点。这里最重要的是，当我在左和右之间切换时，重定时结果是一样的。这是因为 VectorGenerator 同时计算左右运动。

我们可以将这个结果与 Kronos 重新计时进行比较，Kronos 将分别处理左视图和右视图。这是 Kronos 的结果，你可以看到重定时是不同的。当你在立体 3D 中看到它时，这将会很突出，所以为了说明这一点，我将切换到一个浮雕节点。当我们切换到 Kronos 结果时，你可以看到错误跳出来。因此，生成重定时相当简单。如果有任何错误，您可以尝试使用不同的运动矢量生成多个 retimes，并将其组合在一起。您还可以修复一个视图，并使用重新计时的差异将该修复推到另一个视图上。所以，在这里我使用一个 NewView 节点集从左边重建重新计时的权利。我们可以将重建的视图与原始的重定时右视图进行比较，这样你就可以使用重定时视差来帮助构建完全匹配的重定时板。
