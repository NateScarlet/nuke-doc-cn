O_ColourMatcher

O_ColourMatcher 节点使您能够将一个视图的颜色与另一个视图的颜色进行匹配。它是专门设计来处理微妙的 立体视图之间有时存在的颜色差异。

面具

一个可选的遮罩，用于确定从哪里获取颜色分布。例如，如果您有一个在绿色屏幕前显示一个人的剪辑，您可能希望使用遮罩来排除绿色区域，以便节点集中精力匹配该人。

在 基本 模式下，O_ColourMatcher 计算源视图蒙面区域上的变换，但将其应用于它正在更正的整个视图。在 本地匹配 模式下，它计算蒙面区域上的变换，并将其仅应用于该区域。

匹配

MatchWhat

从左到右

设置如何匹配视图之间的颜色:

• 从左到右 -调整左视图的颜色以与右视图的颜色相匹配。

• 从右到左 -调整右侧视图的颜色以与左侧的颜色相匹配。

模式

匹配模式

本地匹配

设置用于颜色匹配的算法:

• 基本 -此模式采用一个整个视图的颜色分布，并修改该颜色分布以匹配另一个视图的分布。

• 本地匹配 -这种模式首先分为两个图像转换成块依 块大小 控制。然后，它将想要修改的视图的颜色分布匹配到同一视图的重建版本，该版本是使用源视图的像素构建的。如果视图之间存在局部颜色差异，例如一个视图比另一个视图亮的高光，这可能会很有用。

注意: 这两种模式都需要 O_DisparityGenerator 节点和 O_ColourMatcher 节点上游的 O_OcclusionDetector 节点。

修正

遮挡校正

闭塞

定义当本地颜色匹配无效时，哪些区域会接收颜色校正:

• 闭塞 -仅填充被遮挡的像素，其他视图中缺少颜色。

• 遮挡和边缘 -填充被遮挡的像素，其中颜色从另一个视图中丢失，并补偿匹配和/或重建可能失败的边缘的视差变化。

• 没有 -不应用遮挡或边缘校正。

调校
