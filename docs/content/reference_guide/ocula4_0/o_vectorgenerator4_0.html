O_VectorGenerator

O_VectorGenerator 生成 立体图像中每个视图的运动矢量场。运动向量场将一帧上像素的位置映射到相邻帧中相应像素的位置。它与图像具有相同的尺寸，但每像素包含一个 (x，y) 偏移。这些偏移显示了如何将相邻图像扭曲到当前图像上。

面具

一个可选遮罩，指定要从运动计算中排除的区域。您可以使用此输入来防止遮挡处的扭曲，或者通过忽略所有前景元素来计算背景层的运动。

请注意，遮罩应该存在于两个视图中，O_VectorGenerator 将 1 的 alpha 值视为前景，并使用附近的向量来重新创建对象边界，而不是图像数据，将其模糊为 0 值。使用 Roto 或 RotoPaint 创建蒙版时，可以使用 羽毛 控件来扩展计算。例如，矢量地图可能在边缘有一个更清晰的过渡，但是在面具上应用羽毛可以帮助平滑结果图像。

启用时，渲染发生在 本地 GPU 指定 (如果可用)，而不是 CPU。GPU 和 CPU 之间的输出在 NVIDIA GPU 上是相同的，但是使用 GPU 可以显著提高处理性能。

Nuke 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上支持 AMD gpu，运行 OS X 10.9.3 “mavericks” 或更高版本。虽然，在某些情况下，GPU 和 CPU 之间的比特级相等保持不变，但对于某些操作，这种配置可能存在精度限制。

注意: 在没有本地 GPU 的情况下启用此选项允许脚本在 GPU 上运行，只要脚本在具有 GPU 可用的机器上打开。

面具

MaskWith

没有

在图像中设置生成向量时要忽略的区域。在 O_VectorGenerator 控件中，设置 面具 到要用作遮罩的组件。以下是 面具 可用设置:

• 没有 -使用整个图像区域。

• 源 Alpha -使用的 alpha 通道 源 剪辑作为面具。

• 源倒阿尔法 -使用的倒阿尔法通道 源 剪辑作为面具。

• 遮罩亮度 -使用的亮度 面具 作为遮罩输入。

• 掩模倒置亮度 -使用倒置的亮度 面具 作为遮罩输入。

• 面具阿尔法 -使用的 alpha 通道 面具 作为遮罩输入。

• 面具倒阿尔法 -使用的倒阿尔法通道 面具 作为遮罩输入。

矢量细节

VectorDetail

1

调整计算的运动矢量的细节。更高的细节可以获得更精细的运动，但需要更长的时间来计算。

强度

强度

5

设置帧之间匹配像素的强度。更高的值允许您准确地将一幅图像中的相似像素与另一幅图像进行匹配，即使生成的运动场参差不齐，您也可以集中精力进行细节匹配。较低的值可能会错过本地细节，但不太可能为您提供奇怪的伪向量，从而产生更平滑的结果。

通常，有必要将这些品质中的一个与另一个交换。您可能希望增加此值以强制图像匹配，例如，在错过细节的地方，或者减少此值以平滑运动向量。

一致性

一致性

1

设置向前和向后向量被迫匹配的程度。增加 一致性 使向前和向后向量彼此更相似，但这可能会导致向量与图像的匹配较少。

平滑度 平滑度 1

在图像匹配后，将额外的平滑应用于运动矢量场作为后处理。值越高，结果越平滑。
