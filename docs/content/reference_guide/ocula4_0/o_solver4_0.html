O_Solver

O_Solver 节点定义输入图像中两个视图之间的几何关系 (即相机关系或求解)。当将素材与 O_VerticalAligner 对齐时，这是必要的。使用中的对齐控制时，还需要计算对齐的视差向量 O_DisparityGenerator 。使用 O_ColourMatcher 和 O_FocusMatcher 进行颜色和焦点匹配不需要 O_Solver 数据。

相机

一个预先跟踪的核弹立体相机，描述了用于拍摄的相机设置 源 图像。这可以是您使用 CameraTracker 节点跟踪的相机，也可以是从第三方相机跟踪应用程序导入到 Nuke 的相机。此输入是可选的。

忽略

一个遮罩，指定在特征检测和分析期间要忽略的区域。这可能是有用的，如果在 源 图像正在产生不正确匹配的特征。此输入是可选的。

源

一对立体图像。这些可以是你想处理的图像，也可以是用相同的相机设置拍摄的另一对图像。

所有键的单一解决方案

SingleSolve

残疾人

启用后，O_Solver 使用您设置的所有关键帧计算单个解析式。将此用于不会随时间变化的钻机，以获得比使用单个关键帧或在一个剪辑上执行 o _ solver 分析然后重新用于另一个剪辑时更准确的结果。

如果对齐存在抖动，或者分离、收敛或缩放发生变化，请不要使用此选项。相反，对每个关键帧使用单独的解析，并在对齐方式更改的位置放置关键点。

• 关键帧匹配 -在查看器覆盖中显示相机关系计算的功能和匹配。仅计算关键帧的特征匹配。

匹配偏移

偏移

100

设置应用于对齐特征匹配的偏移 (以像素为单位)。您可以:

• 增加这个值可以人为地增加差异，这样就更容易看到特征匹配的水平程度。

• 减小此值可将特定匹配项的差异设置为零，并检查每个特征处的垂直偏移。比赛应该放在一起。

注意: 的 匹配偏移 控件仅在 显示 设置为 预览对齐 .
