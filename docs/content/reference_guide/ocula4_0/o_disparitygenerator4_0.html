O_DisparityGenerator

O_DisparityGenerator 节点用于为立体图像创建视差图。视差图描述一个视图中像素的位置，相对于另一个视图中相应像素的位置。它包括两组视差向量: 一组将左视图映射到右，另一组将右视图映射到左。

以下是 Nuke 节点依靠视差图来产生输出:

• O_OcclusionDetector

• O_ColourMatcher

• O_FocusMatcher

• O_verticalalaligner (在 局部对齐 模式)

• O_NewView

• O_InteraxialShifter

• O_DisparityToDepth

• O_DisparityViewer

预设

DisparityType

正常

使用 预设 下拉列表通过更改适当的细化控制自动对差异结果进行调整:

• 自定义 -手动调整控件时自动选择,

• 正常 -所有控件的默认值。

• 强 -增加匹配 强度 之间的图像以牺牲 稳定 .

你可以使用 强 当使用 O_NewView 、 O_ColourMatcher 和 O_FocusMatcher 构建图片效果不佳时。

• 咄咄逼人 -增加 强度 用于重建图像尽可能接近源，但减少 稳定 有利于准确性。

此选项在 O_NewView 中提供了最佳结果，可以从另一个视图中重现一个视图的外观，对于颜色或焦点匹配非常有用。

注意: 激进的计算会产生较差的稳定性，这使得它不适合从一个视图复制到另一个视图。

• 平滑 -减少匹配 强度 之间的图像和集中 稳定 .

此选项产生更平滑、时间稳定的向量。使用 平滑 生成干净的深度图，并在将修复从一个视图复制到另一个视图时防止闪烁。

注意: 从一帧到下一帧的平滑向量更清晰、更稳定，但是使用 O_NewView 可能无法很好地重建图像。

• 对齐 -此选项启用 对齐 控制，这需要一个 O_Solver 上游。

• 快速 -此选项禁用 稳定 控制，减少处理时间。

矢量细节

VectorDetail

1

调整计算的视差向量的密度。更高的细节可以获得更好的视差变化，但需要更长的时间来计算。

强度

强度

5

设置在左视图和右视图之间匹配像素时应用的强度。更高的值允许您准确地将一幅图像中的相似像素匹配到另一幅图像，即使生成的视差图是锯齿状的，您也可以专注于细节匹配。较低的值可能会错过本地细节，但不太可能为您提供奇怪的伪向量，从而产生更平滑的结果。

通常，有必要将这些品质中的一个与另一个交换。你可能想增加 强度 强制视图匹配错过的细节，或者减少细节以平滑视差图。

一致性

一致性

1

设置左视图和右视图中的相同点相互映射的精确程度。增加该值以鼓励左、右视差向量匹配。

平滑度

平滑度

1

控制如何将图像边缘用作场景中深度锐化过渡的线索。值越高，图像边缘深度之间的过渡就越平滑。

对齐

对齐

0

设置约束差异的程度，以匹配上游 O_Solver 节点定义的垂直路线。增加值会强制对齐差异，并且需要至少一个关键帧匹配。

注意: 默认的 对齐 的价值 0 使用无约束运动估计计算视差，因此不需要 o _ solver 数据。

将 O_DisparityGenerator 与 O_Solver 对齐数据结合使用，可以约束生成的视差向量以匹配全局板对齐。如果你的盘子没有太多细节，比如背景中有标记的蓝屏图像，或者像天空这样有很多无特征区域的盘子，你可能会想这样做。

注意: 使用 O_Solver 对齐数据可以减少垂直视差随深度的变化 局部对齐 在 O_VerticalAligner 中，并在 O_NewView 中导致垂直偏移，其中视差不会拾取匹配图像所需的局部垂直偏移。

稳定

稳定

1

设置通过在多个帧 (而不是单个立体帧) 上生成视差来强制向量在时间上一致的程度。增加该值会产生更平滑的向量，随着时间的推移，这些向量会更稳定，但会以增加处理时间为代价。
