安装后，所有 Nuke 应用程序通过桌面图标、浏览器或使用参数从终端运行。

Nuke 系统要求

合格的操作系统

• CentOS/RHEL 5

• CentOS/RHEL 6

注意: 其他操作系统可能与 Nuke ,但尚未完全测试。如果您对特定操作系统有任何问题，请联系我们的支持团队。

其他系统要求

• X86-64 处理器，如英特尔奔腾 4 或 AMD 速龙，支持 SSE3 指令集 (或更新)。

• 可用于缓存和临时文件的 5 GB 磁盘空间。

• 至少 8gb 的 RAM。

• 具有至少 1280x1024 像素分辨率和 24 位颜色的显示器。

• 显卡具有至少 512 MB 的视频内存和对 OpenGL 2.0 的驱动程序支持 (最低要求)。

• 要启用查看器处理的可选 GPU 加速，您需要支持浮点纹理和 GLSL 的 OpenGL 2.0。

• 要启用 Nuke 要使用 GPU 计算某些节点，还有一些额外的要求。有关更多信息，请参见 GPU 加速要求 .

• R3D 火箭卡需要火箭驱动程序 1.4.19.0 和固件 1.1.16.5 或更高版本。

注意: 为了避免图形问题，如文本在查看器和节点图形中消失，保持显卡驱动程序的最新状态非常重要。驱动程序更新可以从显卡制造商的网站获得 (例如, Www.nvidia.com 和 Support.amd.com )。

注意: 如果您正在使用 R3D Rocket 显卡，请注意在 Nuke 当你以全分辨率阅读时，很可能只会更快。例如，如果你以半分辨率阅读，使用 Nuke 没有启用 R3D 火箭卡可能会更快。这是因为 R3D Rocket 显卡在同时读取多个帧时速度很快。这不是如何 Nuke 在内部工作，因此在禁用 R3D 火箭卡的情况下，在较低分辨率 (<4k 宽度) 下工作时，读取速度有时可能会更快。请注意，R3D 火箭卡总是产生比 Nuke 下采样时。此外，R3D 火箭卡一次只能由一个应用程序使用，所以如果您正在查看多个 Nuke 脚本一次，你可能只能在一个中使用 R3D 火箭卡。

GPU 加速要求

如果你想启用 Nuke 要使用 GPU 计算某些节点，还有一些额外的要求。你需要有:

• 计算能力 2.0 (Fermi) 或以上的 NVIDIA GPU。NVIDIA gpu 的计算功能列表可从以下网址获得:
Www.nvidia.co.uk/object/cuda_gpus_uk.html

注意: 计算能力是 GPU 硬件的特性，不能被软件更新改变。

• 能够运行 CUDA 4.2 或以上的图形驱动程序。这些与 NVIDIA GPU 的常规驱动程序捆绑在一起。从 2012年4月起，司机支持 CUDA 4.2。

转到 Http://www.nvidia.com/Download/Find.aspx？lang=en-us 欲了解更多信息。

注意: 如果您的计算机进入睡眠模式，CUDA 驱动程序无法恢复，您必须重新启动 Nuke 使用 GPU 加速。

提示: 我们建议尽可能使用最新的图形驱动程序。
