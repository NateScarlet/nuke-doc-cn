输入进程和查看器进程控件

输入过程 和 浏览器进程 操作可用于在图像显示在显示器上之前修改 “已查看” 节点中的图像。两者都只影响激活它们的查看器，而不影响渲染的输出。输入过程是一个遗留系统，它使用节点图中实例化的节点来处理图像。这对于特定于脚本的、临时的或实验性的使用非常方便，但由于节点意外被删除或更改，并且仅限于单个节点，因此可能容易出错。添加了查看器处理系统 Nuke 工作室 5.2，并允许在启动时从 Python 编程语言注册 gizmo (或编译节点)。注册的项目出现在查看器的下拉菜单中，当选择项目时，该节点在查看器内部被实例化，因此没有意外删除或修改的危险。这也使多个浏览器进程可以在不同的启动点注册 (如 Nuke 工作室 通过 NUKE_PATH 菜单工作。py 文件)。

查看器设置包含一个选项，用于在查看器进程之前或之后应用输入进程，因此两者可以结合使用，例如, 在输入过程中应用投影蒙版后，观众过程应用电影外观轮廓。虽然您可以将两者合并到一个查看器进程节点中，但将操作分开可能是有利的。同时提供查看器过程和输入过程提供了很大的灵活性。

可以通过在节点图中创建节点并将其命名为输入进程来创建输入进程。 Nuke 工作室 的 编辑 菜单。输入进程命名后，IP 按钮将出现在查看器控件中。激活 IP 按钮后，您查看的任何图像都将通过输入过程传递。

与输入进程不同，查看器进程是使用 Python 注册的。它们可以独立于会话，并且总是出现在查看器的查看器进程下拉菜单中。有两个预定义的查看器进程, SRGB 和 Rec709 ,但您也可以构建和添加自己的。从 “查看器进程” 下拉菜单中选择查看器进程后，您查看的任何图像都将通过该查看器进程。

只要有可能，输入进程和查看器进程都会在 GPU 上执行。1D LUT 和 3D LUT (Vectorfield) 有 GPU 实现，因此内置的查看器进程在 GPU 上运行 (除非 Gl 缓冲区深度 已设置为 字节 在查看器设置中，在这种情况下，所有处理都在 CPU 上完成)。要获得用于自定义查看器过程 gizmo 的节点的 GPU 版本，请按 x 在节点图上，输入 ViewerGain , ViewerGamma ,或 ViewerClipTest 在命令输入窗口中，然后按 返回 .

注意: Note that Input Processes and Viewer Processes are part of a built-in, fixed pipeline of nodes that are applied to images before they are displayed in the Viewer. This pipeline is either:
gain > Input Process > Viewer Process > gamma > dither > channels > cliptest (if 输入过程顺序 has been set to 在查看器进程之前 in the Viewer settings)
OR
gain > Viewer Process > Input Process > gamma > dither > channels > cliptest (if input process order has been set to 浏览器进程后 in the Viewer settings).
However, depending on what the Input Process and Viewer Process are doing, the order in the built-in pipeline may not be the correct order. Therefore, if your Input Process or Viewer Process have controls that also exist for the Viewer, such as float controls named 增益 , 伽马 , or 快船测试 , then the Viewer drives them from the corresponding Viewer controls and does not do that image processing itself. This allows you to implement these controls in your Input Process or Viewer Process node/gizmo using whatever nodes and order you want. If your Input Process and Viewer Process do not have these controls, then the Viewer applies the effects in its normal way according to the built-in pipeline.
In the built-in pipeline, dither is applied to diffuse round-off errors in conversion of floating point data to the actual display bit depth. Although the cliptest is drawn at the end, it is computed on the image as input to the Viewer.

注意: 默认情况下，预定义的查看器进程, SRGB 和 Rec709 ,影响所有通道。但是，如果您希望它们只影响红色、绿色和蓝色通道，您可以激活 仅将 LUT 应用于彩色通道 在单个查看器设置中或在 观众 首选项的选项卡。

输入过程控制

要激活或停用输入过程的效果，请单击查看器控件中的 IP 按钮。请注意，IP 按钮仅在 输入过程 查看器设置中的字段不为空。仅当节点图中的节点设置为输入进程时，才会启用该按钮。

切换输入过程的使用。

要打开查看器设置，请按 S 在查看器上，或选择 查看器设置 从查看器的右击菜单中。默认情况下, 输入过程 设置为 VIEWER_INPUT 。如果节点图中存在名为 VIEWER_INPUT 的节点，它将自动用作查看器的输入过程。这确保了与 pre-5.2 脚本的向后兼容性。

但是，输入进程节点不必命名为 VIEWER_INPUT。您可以使用任何节点作为输入过程。执行以下操作:

1。 在节点图中选择节点并选择 编辑 > 节点 > 用作输入过程 .

或者，您可以按 S 在查看器上打开查看器设置，并在 输入过程 字段。

2. 在查看器设置中，您还可以定义在当前使用的查看器进程之前还是之后应用输入进程。要这样做，设置 输入过程顺序 要么到 在查看器进程之前 或 浏览器进程后 .

输入进程节点不应连接到节点图中的其他节点。如果尝试连接它，查看器中将显示错误。如果从节点图中删除输入进程节点，则禁用输入进程的效果。

查看器进程控件

要激活查看器进程，请从查看器右上角的查看器进程下拉菜单中选择它。您现在使用此查看器查看的任何图像都将通过选定的查看器进程进行传递。

Nuke 工作室 包括以下预定义的查看器进程小控件: SRGB 和 Rec709 。默认情况下，使用 sRGB 是因为它是典型计算机监视器的最佳猜测默认值。

除了使用预定义的查看器进程之外，您还可以通过将节点或 gizmo 注册为查看器进程来添加自己的进程。您可以使用自定义查看器 LUTs 注册任意数量的查看器进程。 有关创建和注册自定义查看器进程的更多信息，请参见 创建自定义查看器进程 在 配置 Nuke 第二章 Nuke 工作室 用户指南 .

所有可用的查看器进程 (包括预定义进程和自定义进程) 都出现在查看器控件的查看器进程下拉菜单中。要禁用查看器进程的使用，请选择 没有 从查看器进程下拉菜单中。

要打开当前活动的查看器进程的 “属性” 面板，请选择 显示面板 从查看器进程下拉菜单中。

请注意，如果要调整的控件与任何查看器控件具有相同的名称 (例如, 增益 或 伽马 )，您应该调整查看器上的控件。这将驱动以下控件:

• 输入进程控制输入进程是否正在使用

• 在查看器进程的控件中，如果没有使用输入进程。

提示: 如果要渲染包含查看器进程效果的文件，可以选择 编辑 > 节点 > 将查看器进程复制到节点图 在节点图中创建查看器进程节点的实例。
