参数

此插件的参数如下所述。

钻机区域 -定义要修复的区域。

• 盒子 -修复矩形框内的区域，由下面的框参数或屏幕框控制。

• Src 阿尔法 -修复由源输入的 alpha 定义的区域。

• Src 倒 α -修复由源输入的反向 alpha 定义的区域。

• RigMask 亮度 -修复由亮度定义的区域 钻机面罩 输入。

• RigMask 倒亮度 -修复由反向亮度定义的区域 钻机面罩 输入。

• RigMask Alpha -修复由 alpha 定义的区域 钻机面罩 输入。

• RigMask 倒阿尔法 -修复由倒阿尔法定义的区域 钻机面罩 输入。

帧搜索 -设置是向前、向后或双向搜索以查找丢失的数据。

• 向前和向后 -在当前帧之前和之后进行搜索。

• 向前 -搜索当前帧后的帧。

• 向后 -在当前帧之前搜索帧。

帧范围 -设置算法在序列中向前和向后查找丢失数据的帧数。如果你得到红色像素，那么增加这个值。请参阅 快速启动 .

范围内使用的帧 -如果 帧范围 必须设置为大量才能进行有效修复，渲染时间可能会令人望而却步。 范围内使用的帧 可以通过不使用每一帧填充前景区域来加快修复速度，有效地跳过帧。然而，这可能会降低结果的质量。

• 所有帧 -使用搜索帧范围中的每一帧来构建修复。

• 一半的帧 -使用搜索帧范围内的所有其他帧来构建修复。

• 帧的四分之一 -使用搜索帧范围中的每四帧来构建修复。

• 10% 的框架 -使用搜索帧范围内的每十分之一帧来构建修复。

• 最大 25 帧 -使用不超过 25 帧的搜索帧范围来构建修复。这个选项可以是很有用的。 帧范围 已设置为一个非常大的数字。

最大钻机运动 -为了避免透视变化，F_RigRemoval 在钻机区域周围的区域内搜索丢失的数据。 最大钻机运动 定义此区域的宽度 (以像素为单位)。在快速运动 Src 镜头需要比慢速移动更高的值。然而，因为用于修复的区域可能来自图像的不同部分，所以高值可能会引入透视问题。

钻机区域框 -用于定义修复的矩形区域 钻机区域 设置为 盒子 .

钻机区 BL -控制装备区域左下角的位置。

钻机区域 TR -控制装备区域右上角的位置。

高级 -较少使用的细化控制。

过滤 -设置过滤质量。

• 低 -低质量，但渲染速度快。

• 介质 -使用双线性过滤器。这提供了良好的结果，并且比高过滤更快地渲染。

• 高 -使用 sinc 滤波器来插入像素，从而进行更清晰的修复。这给出了最好的结果，但需要更长的时间来处理。

亮度正确 -打开该开关，以纠正来自其他帧的信息的亮度变化。如果照明在整个序列中发生变化，这一点尤其重要。

透视正确 -打开此开关以纠正细微的透视变化。

重叠正确 -修复是使用序列中其他帧的信息片来构建的。这些切片可以重叠和混合，以提供更自然的修复。此参数控制区域重叠的程度。过多增加此参数会降低图像清晰度。

修复失败标记不透明度 -设置用于显示修复失败位置的红色像素的透明度级别。

保存阿尔法 -打开该开关以保留原始 alpha 通道。默认情况下，这是关闭的，alpha 通道设置为白色，其中修复失败，其他地方都设置为黑色。
