向元数据添加时间代码

向元数据添加时间代码:

1。 选择 元数据 > AddTimeCode 将 AddTimeCode 节点插入到节点树中。

将时间码添加到正在传递的元数据中。默认情况下，第一帧的时间代码是 01: 00。它会根据输入剪辑的播放速度在整个帧范围内更新，这反过来又由 Fps (每秒帧数) 项目设置中的参数。如果你改变 Fps 值，元数据中的时间代码将更新以反映更改。

Nuke 还可以处理丢弃帧，例如当剪辑的帧速率为 29.97 或 59.94 时。而不是默认的 HH: MM: SS: FF 时间码格式，使用格式 HH; MM; SS; FF，由分隔 ; (分号)。

注意: 使用带有非丢弃帧时间代码的分号分隔符在查看器中显示错误。

2. 如果不希望开始帧上的时间码为 01: 00，请在 开始代码 字段。
3. 如果要手动指定播放速度，而不是从元数据和项目设置中获取，请取消选中 从元数据获取 FPS 而进入一个新的价值 Fps 字段。
4. 如果要指定不同于第一帧的开始帧，请选中 使用开始帧？ 而进入一个新的价值 开始帧 字段。

如果要在图像上显示时间码，请在 AddTimeCode 节点后插入一个文本节点，然后输入 [时间码] 在 消息 字段。有关通过表达式引用元数据的更多信息，请参见 使用 Tcl 表达式访问元数据 .
