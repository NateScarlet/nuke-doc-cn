核武器 环境变量

下表列出了环境变量 核武器 识别。

环境变量

描述

FN_CRASH_DUMP_PATH

允许您指定默认情况下保存问题报告程序转储的位置。

FN_DISABLE_LICENSE_DIALOG

或

FN_NUKE_DISABLE_TMPLIC_NOTIFY_DIALOG

默认情况下，如果您安装了临时许可证, 核武器 在启动时显示一个对话框，提醒您剩余天数。如果要禁用此行为，可以将这些环境变量中的任何一个设置为 1，以抑制关于即将到期的许可证的警告消息。

注意: 如果找不到许可证，您仍然会收到警告，例如，如果您只有 核武器 许可证，但你试图运行 NukeX .

FN_LICENSE_DIALOG_DAYS_LEFT_BEFORE_PROMPT

默认情况下，如果您安装了临时许可证, 核武器 在启动时显示一个对话框，提醒您剩余天数。如果要在到期前禁用此行为，可以将此环境变量设置为所需的天数。

注意: 如果找不到许可证，您仍然会收到警告，例如，如果您只有 核武器 许可证，但你试图运行 NukeX .

FN_NUKE_DISABLE_GPU_ACCELERATION

此变量禁用 核武器 的 CUDA 和 OpenCL 功能。启用时，任何本地安装的 gpu 都将被禁用，无法从 偏好 > 性能 > 硬件 > 默认闪烁设备 下拉列表。任何 GPU 加速节点，如 Kronos 和去噪，默认在 CPU 上处理。

FN_SUBSCRIPTION_LICENSE_DIR

在 Windows 上，包含非 ASCII 字符的用户名可能会导致订阅授权失败。如果显示类似以下内容的许可错误:

无法创建订阅许可证目录: C: \ Users \ zoe è hern á ndez \ FoundryLicensing \

尝试使用此环境变量将许可证目录更改为备用位置。

Foundry_LICENSE

的位置 核武器 RLM 许可证文件，如果不使用以下推荐位置:

在 Mac OS X 和 Linux 上 :

/Usr/本地/铸造/RLM

在 Windows 上:

驱动器号 : \ 程序文件 \ 铸造 \ RLM

注意: 我们正在将许可证从 FLEXlm 迁移到 RLM 许可模式。如果你有兴趣提前申请 RLM 许可，请联系 Sales@foundry.com 获得替换许可证。

FOUNDRY_LICENSE_DEBUG

此变量将附加许可信息打印到命令行或终端。

FOUNDRY_LICENSE_FILE

的位置 核武器 FLEXlm 许可证文件，如果不使用以下推荐位置:

在 Mac OS X 和 Linux 上 :

/Usr/本地/铸造/FLEXlm

在 Windows 上:

驱动器号 : \ Program Files \ Foundry \ FLEXlm

注意: 我们正在将许可证从 FLEXlm 迁移到 RLM 许可模式。如果你有兴趣提前申请 RLM 许可，请联系 Sales@foundry.com 获得替换许可证。

FOUNDRY_LOG_FILE

此变量指定的位置 核武器 的日志文件。如果不指定日志文件，所有输出都将显示在屏幕上。

FOUNDRY_LOG_LEVEL

此变量设置日志记录的级别 核武器 在操作过程中产生。有四个细节级别，从最小到详细的滑动比例:

• 错误

• 警告

• 消息

• 详细

注意: 当指定 FOUNDRY_LOG_FILE 时，将日志记录级别设置为 verbose 可以生成大型日志文件。

FRAMESERVER_LOG_DIR

如果您想将日志文件与默认的 NUKE_TEMP_DIR 分开，此变量用于为帧服务器指定一个不同的位置。

请参阅 在外部机器上使用帧服务器 欲了解更多信息。

HIERO_DISABLE_THUMBNAILS

将此变量设置为停止 核武器工作室 正在加载缩略图。

HIERO_DISABLE_THUMBNAILS_CACHE

将此变量设置为停止 核武器工作室 加载后缓存缩略图以改进访问。

注意: 此变量不清除缓存，必须手动删除缓存文件。

NUKE_AJA_CHANNEL

AJA 卡采用 3g 级信号 (主要是 12 位 444 RGB)，并将其组合成一个 3G-B (B 表示 B 级，因此 3G-B) 流，默认情况下通过 SDI1。使用这些环境变量自定义此输出行为:

• NUKE_AJA_CHANNEL -将此变量设置为 2 、 3 或 4，以通过 SDI2 、 SDI3 或 sdi4 输出单个流。

• NUKE_AJA_DUALOUTPUT -将此环境变量设置为 1，以强制卡通过 SDI1 和 sdi2 将单个 3g 流分成两个 1.5G 流。

结合这两个环境变量可以强制流通过备用 SDI 输出进行拆分和输出。例如:

• 双输出通道 = 1 或通道 = 2 导致来自 SDI1 和 sdi2 的两个 1.5G 流。

• 双输出通道 = 3 或通道 = 4 导致来自 SDI3 和 sdi4 的两个 1.5G 流。

注意: 某些模式，如 12 位 444，需要 3g 流。否则，该卡将使用指定通道号上的单个流。

NUKE_AJA_DUALOUTPUT

NUKE_ALLOW_GIZMO_SAVING

核武器不允许你 覆盖 和 另存为 默认情况下，不将 gizmo 复制到组。将此环境变量设置为 1 将启用此行为，因此艺术家在编辑 gizmo 之前不需要复制它。

NUKE_CRASH_HANDLING

Breakpad 崩溃报告允许您在不太可能发生崩溃的情况下向 Foundry 提交崩溃转储。默认情况下，在 GUI 模式下启用崩溃报告，在终端模式下禁用崩溃报告。

当 NUKE_CRASH_HANDLING 设置为 1 时，会在 GUI 和终端模式下启用崩溃报告。

当 NUKE_CRASH_HANDLING 设置为 0 时，在 GUI 和终端模式下都禁用崩溃报告。

NUKE_DEBUG_IMAGECACHE

启用后，Comp Viewer 图像缓存数据将打印到命令行或终端。显示有关使用的磁盘空间、缓存的文件数量和缓存位置的信息。

NUKE_DEBUG_MEMORY

当处理大图像时, 核武器 可能需要在渲染期间释放内存。当这种情况发生并且 NUKE_DEBUG_MEMORY 设置为 1 时, 核武器 将以下信息打印到控制台:

内存: 超过最大使用量，试图将使用量从 1 GB 减少到 924 MB。

如果未设置此变量，则无法看到调试内存消息。

请注意，这里，KB 、 MB 、 GB 和 TB 的平均值为 1000。例如，1 MB 表示 1,000,000 字节。

NUKE_DISK_CACHE

位置在哪里 核武器 保存查看器中显示的所有最近图像。理想情况下，这应该是一个访问时间最快的本地磁盘。

NUKE_DISK_CACHE_GB

磁盘缓存可以达到的最大大小 (千兆字节)。

NUKE_EXR_TEMP_DIR

在 Linux 上，这是位置 核武器 读取 PIZ 时使用临时文件-压缩 。Exr 文件。此环境变量仅在 Linux 上相关。

如果未设置此变量，位置由 NUKE_TEMP_DIR 确定。

NUKE_FONT_PATH

那个位置 核武器 打开 “文本节点属性” 面板时，检查可用字体文件。

NUKE_IGNORE_ROTO_INCOMPATIBILITY

此变量禁用打开包含预 Nuke 8 RotoPaint 节点的脚本时显示的警告对话框。

NUKE_INTERACTIVE

的 进口核武器 默认情况下，函数签出 nuke_r 渲染许可证。如果你想使用 核武器 以交互方式，并且您有交互式许可证，将此环境变量设置为 1。

请参阅 Nuke 作为 Python 模块 欲了解更多信息。

NUKE_LEGACY_CHANNEL_SORTING

此变量禁用新的通道排序行为，其中首先对 RGBA 层进行排序。启用此变量原因 核武器 按字母顺序对通道进行排序。

NUKE_MOV64READER_ENABLE

将此变量设置为 0 以禁用 核武器 的 64 位 mov 解码和回落到 32 位解码。

NUKE_NO_CRASH_PROMPT

当在 GUI 模式下启用崩溃处理时，这允许您控制报告是否自动提交:

当 NUKE_NO_CRASH_PROMPT 设置为 1 时，崩溃报告将自动提交，而不显示崩溃报告对话框。

当 NUKE_NO_CRASH_PROMPT 设置为 0 时, 核武器 在提交崩溃报告之前，始终显示崩溃报告对话框。

NUKE_NO_VIEWER_GPU

禁用 核武器 的 Comp Viewer OpenGL 硬件加速。

NUKE_PATH

与文件相关的位置 核武器 存储自定义。有关更多信息，请参见 加载 Gizmos 、 NDK 插件以及 Python 和 Tcl 脚本 .

NUKE_TEMP_DIR

位置在哪里 核武器 保存没有为其定义特定位置的任何临时文件。

NUKE_WINDOWMANAGER_DEBUG

启用时，数据来自 核武器 的窗口管理器打印到命令行或终端。

TIMELINE_DISABLE_PBO_UPLOADS

启用后，使用像素缓冲区对象 (PBO) 将纹理从 RAM 上传到 GPU 的性能优势将被禁用。

如果您注意到播放降级，可以尝试禁用 PBOs。

OCIO

将此变量设置为用于颜色转换的 OCIO 配置文件的位置。

注意: 如果计划使用首选项中指定的 OCIO 配置文件，请确保 偏好 > 项目默认值 > 色彩管理 > 出口 > 导出到 Comp 时使用 OCIO 节点 复选框已启用。

OFX_PLUGIN_PATH

位置在哪里 核武器 查找 OFX 插件。有关更多信息，请参见 正在加载 OFX 插件 .
