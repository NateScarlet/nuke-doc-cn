跨平台处理文件路径

如果您的设施使用 Nuke 在几个操作系统上，您可能需要配置 Nuke 替换文件路径的开头，以便在一个平台上创建的脚本也可以在另一个平台上运行。

例如，为了确保在 Windows 上创建的文件路径也可以在 Linux 上运行，反之亦然，您可以执行以下操作:

1。 创建一个名为 Init.py 在你的插件路径目录中，如果一个已经不存在。

有关插件路径目录的更多信息，请参见 加载 Gizmos 、 NDK 插件以及 Python 和 Tcl 脚本 .

2. 打开 Init.py 在文本编辑器中添加文件并添加以下格式的条目: 

导入平台

Def filenameFix (文件名):
如果 (“Windows” 、 “Microsoft”) 中的 platform.system ():
返回文件名。替换 (“/SharedDisk/”，“p: \”)
返回文件名。替换 (“p: \”，“/SharedDisk/”)

这样，Windows 文件路径 (从 P: \ 在上面的例子中) 被替换为 Linux 文件路径 (从 /SharedDisk/ ) 在引擎盖下，每当 Nuke 脚本在 Linux 上使用。否则，将使用 Windows 文件路径。

请注意，图形用户界面 (GUI) 中显示的文件路径不会更改。如果您正在使用 P: \ 在节点控件中，仍显示为 P: \ 。然而，在 Linux 上, Nuke 解释 P: \ 作为 /SharedDisk/ .
