定义 Nuke 插件路径

要定义插件路径:

1。 在每个艺术家的机器上，创建一个名为 NUKE_PATH .
2. 分配 NUKE_PATH 与文件相关的目录的路径名的环境变量 Nuke 定制将驻留。

例如，在使用 csh 或 tcsh shell 的 Mac OS X 上:

Setenv NUKE_PATH/SharedDisk/Nuke

或者，如果您使用的是 bash 或 ksh shell:

导出 NUKE_PATH =/SharedDisk/Nuke

加载插件 (plugin_find ()) 搜索 NUKE_PATH，直到找到第一个插件，确保只加载最多的本地插件。例如，如果 NUKE_PATH 变量包含:

Project_dir: studio_dir: company_dir

按照以下顺序搜索路径，直到找到第一个插件:

• ~/。 Nuke

• Project_dir

• Studio_dir

• Company_dir

• Nuke_dir

然而，在加载时，NUKE_PATH 环境变量会以相反的顺序进行分析 Init.py 和 Menu.py 所有发现的副本都被使用。这允许本地化设置覆盖更多的全局设置。因此，使用上面的目录层次结构, Init.py 脚本将按如下方式执行:

• Nuke_dir/init.py

• Company_dir/init.py

• Studio_dir/init.py

• Project_dir/init.py

• ~/。Nuke/init.py
