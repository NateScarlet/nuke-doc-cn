命令行操作

启动时，命令行标志激活各种选项 Nuke 从命令行或终端，并提供附加功能 Nuke 。首先让我们讨论如何启动 Nuke 从一个外壳。

在窗户上

注意: 如果您使用的是终端模拟器，请按照 Mac OS X 或 Linux 下的步骤操作，根据需要替换文件路径。

打开命令行提示并更改目录，如下所示:

光盘 C: \ 程序文件 \ Nuke 10.5v8 \

要启动 Nuke ,键入以下命令:

Nuke 10.5 。Exe

或者，您可以设置 道奇 来点, Nuke 然后你可以发射 Nuke 从任何目录:

Doskey nuke = "C: \ Program Files \ Nuke 10.5v8 \ Nuke 10.5 。Exe"

如果你想这样做 NukeX ,输入:

Doskey nukex = "C: \ Program Files \ Nuke 10.5v8 \ Nuke 10.5 。Exe ”-- nukex

如果你想这样做 Nuke 工作室 ,输入:

Doskey nukes = "C: \ 程序文件 \ Nuke 10.5v8 \ Nuke 10.5 。Exe ”-- 工作室

在 Mac OS X 上

Open a command line prompt and change directory as follows:

光盘/应用程序/ Nuke 10.5v8 / Nuke 10.5v8 。应用程序/内容/MacOS/

To launch Nuke , type this command:

。/ Nuke 10.5v8

或者，您可以将别名设置为指向 Nuke 然后你可以发射 Nuke 从任何目录。此过程取决于您的默认 shell 是什么。要获取正在使用的外壳的名称，请启动终端并输入 Echo $ SHELL .

如果您使用的是 bash shell，请输入:

别名 nuke = '/应用程序/ Nuke 10.5v8 / Nuke 10.5v8 。应用程序/内容/MacOS/ Nuke 10.5v8 '

或者，如果使用 tcsh shell，请输入:

别名 nuke/应用程序/ Nuke 10.5v8 / Nuke 10.5v8 。应用程序/内容/MacOS/ Nuke 10.5v8

如果你想别名 NukeX ,输入:

别名 nukex/应用程序/ Nuke 10.5v8 / Nuke 10.5v8 。应用程序/内容/MacOS/ Nuke 10.5v8 -- Nukex

如果你想别名 Nuke 工作室 ,输入:

别名 Nuke /应用程序/ Nuke 10.5v8 / Nuke 10.5v8 。应用程序/内容/MacOS/ Nuke 10.5v8 -- 工作室

提示: 您可以向 。Cshrc 或 。Bashrc 文件在主目录中，以便每次打开 shell 时激活它们。有关设置此设置的帮助，请与系统管理员联系。

在 Linux 上

Open a command line prompt and change directory as follows:

Cd/usr/local/ Nuke 10.5v8 /

To launch Nuke , type this command:

。/ Nuke 10.5

Alternatively, you can set an alias to point to Nuke and then you can launch Nuke from any directory. The procedure for this depends on what your default shell is. To get the name of the shell you are using, launch Terminal and enter echo $SHELL .

If you are using a bash shell, enter:

别名 nuke = '/usr/local/ Nuke 10.5v8 / Nuke 10.5 '

Alternatively, if you are using a tcsh shell, enter:

别名 nuke =/usr/local/ Nuke 10.5v8 / Nuke 10.5

If you want to alias NukeX , enter:

别名 nukex =/usr/local/ Nuke 10.5v8 / Nuke 10.5 -- Nukex

If you want to alias Nuke Studio , enter:

别名 nukes =/usr/local/ Nuke 10.5v8 / Nuke 10.5 -- 工作室

TIP: You can add aliases to a .cshrc or .bashrc file in your home directory so that they are activated each time you open a shell. See your Systems Administrator for help setting this up.

使用命令行标志

现在，您可以在启动时开始尝试命令行标志 Nuke 。这里有一个显示版本号和构建日期。

Nuke 版本

如果你有一个 。Nk 脚本，您可以在命令行上呈现它，而无需打开 GUI 版本。这里有一个例子，它渲染了数百帧 Nuke 脚本:

Nuke-F 1-100-x myscript.nk

请注意如何使用 -F 在命令行上切换以指示帧范围，用破折号分隔开始帧和结束帧。

注意: 我们建议您使用 -F 在命令行上定义帧范围时切换，该范围必须位于脚本名称参数之前。
然而，为了向后兼容，你也可以使用旧的语法。为此，将帧范围放在命令的末尾，并使用逗号分隔开始帧和结束帧。例如:
Nuke-x myscript.nk 1,100

要显示可用的命令行标志 (开关) 列表，请使用以下命令:

Nuke -帮助

-- 重新映射

允许您重新映射文件路径，以便轻松共享 Nuke 跨不同操作系统的项目。这是命令行，相当于设置 路径重映射 控制在 偏好 对话框。

-Remap 标志采用逗号分隔的路径列表作为参数。路径成对排列，每对的第一个路径映射到每对的第二个路径。例如，如果您使用:

Nuke-t-重新映射 “X:/path，Y:，A:，B:/anotherpath”

• 任何以开始的路径 X:/路径 转换为从 Y:

• 任何以开始的路径 答: 转换为从 B:/另一个路径

--重新映射国旗抛出一个错误:

• 它是在启动 GUI 模式时定义的，也就是说，没有 -X 或 -T

• 路径不成对。例如，如果您使用:

Nuke-t-重新映射 “X:/path，Y:，A:”

答: 不映射到任何东西，并产生错误。

如果你没有给它任何路径，-remap 标志会给出警告 (但不会出错)。例如:

Nuke-t-重新映射 “”

注意: 请注意，映射仅应用于 Nuke 正在启动的会话。它们不会影响 GUI 使用的 Preferences.nk 文件。

-S #

以字节为单位，设置每个线程的堆栈大小。默认值为 16777216 (16 MB)。允许的最小值是 1048576 (1 MB)。

Nuke 的默认节点都不需要超过默认内存堆栈值，但是如果您编写了一个自定义节点，从内存缓冲区请求一个大堆栈, 增加堆栈大小可以防止堆栈溢出错误。

一般语法

这是启动时使用这些选项的一般语法 Nuke 在命令提示符下:

Nuke <开关> <脚本> <argv> <范围>

<开关>-修改的行为 Nuke 从命令行运行时。上表给出了开关列表。这些有时被称为旗帜。

<脚本>-的名称 Nuke 脚本。

<Argv>-可用于的可选参数 Nuke 。请参见下面的示例。

<Ranges>-这是您想要渲染的帧范围。

例子

让我们考虑一些实际例子。

要启动 Nuke 打开一个脚本。

核弹 myscript.nk

我知道很疯狂，但是我给我的脚本打电话了，-myscript.nk，文件名开头的连字符混淆了 Nuke 。要解决这个问题，如果你不想重命名你的文件，请使用双连字符语法:

核弹-我的剧本

查看图像

要显示图像:

Nuke-v polarbear.tif

要显示多个图像:

Nuke-v polarbear.tif whiteflower.psd mountains.cin

要显示图像序列 (taxi.0001.tif，taxi.0002.tif，.，taxi.0050.tif):

Nuke-v taxi.#.tif 1-50

在命令行上渲染

渲染帧 5 的步骤 Nuke 脚本:

Nuke-F 5-x myscript.nk

渲染 30 到 50 帧的步骤 Nuke 脚本:

Nuke-F 30-50-x myscript.nk

渲染两个帧范围，10-20 和 34-60, Nuke 脚本:

Nuke-F 10-20-F 34-60-x myscript.nk

渲染每 10 帧的 50 帧序列 Nuke 脚本:

Nuke-F 1-50x10-x myscript.nk

这将呈现帧 1 、 11 、 21 、 31 、 41。

在具有两个名为 WriteBlur 和 WriteInvert 的写入节点的脚本中，此命令仅从 WriteBlur 节点渲染 1 到 20 帧:

Nuke-X WriteBlur myscript.nk 1-20

使用 [argv 0]

让我们使用 [argv] 来改变输出文件名。启动的 GUI 版本 Nuke 并创建一个节点树，将检查器放入写入节点。通过双击打开 “写入节点” 属性面板，并在文件文本字段中输入此文件名:

[Argv 0].#.tif

保存脚本并退出 Nuke 。在命令行上键入:

Nuke-x myscript.nk mychecker 1-5

这将呈现 5 帧 (mychecker.0001.tif，mychecker.0002.tif 等)。

您可以添加另一个变量来控制输出图像文件类型。文件文本字段需要:

[Argv 0]。#。[argv 1]

然后使用以下命令呈现脚本:

Nuke-x myscript.nk mychecker cin 1-5

获取 mychecker.0001.cin 、 mychecker.0002.cin 等。

<Argv> 字符串可以是任何 [argv n] 表达式，为脚本提供变量参数。这些必须放在 <脚本> 以及 <范围> 在命令行上。可以包括多个表达式，但每个表达式必须以非数字字符开头，以避免与帧范围控件混淆。有关表达式的更多信息，请参见 表情 .

使用 Python 将 tiff 转换为 jpeg

此命令行方法将 5 个 TIFF 帧转换为 JPEG。

Nuke -t

> R = nuke.nodes.Read (文件 = "myimage.#.tif")

> W = nuke.nodes.Write (文件 = ”myimage.####.jpg ”)

> W.setInput (0，r)

> Nuke.exe 可爱 (“Write1”，1,5)

> 退出 ()

一行一行地输入这些命令有点乏味。所以让我们把它们放在一个名为 imageconvert.py 的文本文件中，然后获取 Nuke 执行 Python 脚本。

猫 imageconvert.py

R = nuke.nodes.Read (文件 = "myimage.#.tif")

W = nuke.nodes.Write (文件 = ”myimage.####.jpg ”)

W.setInput (0，r)

Nuke.exe 可爱 (“Write1”，1,5)

Nuke-t <imageconvert.py

您也可以将 Python 脚本作为命令行参数传递。这样做可以在要传递到脚本的脚本名称之后输入其他参数。当您这样做时，请注意，sys.argv [0] 是正在执行的 Python 脚本的名称，argv [1:] 是您传入的其他参数。下面是一个例子。有关解析参数的其他方法，请参见标准 Python 模块 optparse。

Cat imageconvertwithargs.py

导入 sys

R = nuke.nodes.Read (文件 = sys.argv [1])

W = nuke.nodes.Write (文件 = sys.argv [2])

W.setInput (0，r)

Nuke.exe 可爱 (“Write1”，1,5)

Nuke-t imageconvertwithargs.py myimage.#.tif myimage.####.jpg
