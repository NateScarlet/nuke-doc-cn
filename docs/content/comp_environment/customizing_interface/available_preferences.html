可用的首选项设置

的 偏好 对话框分为以下选项卡:

• 偏好 -文件自动备份、本地文件缓存、内存使用、 GPU 设备和文件路径重放的设置。

• 窗户 -窗口位置、工具提示、窗口捕捉、脚本加载、文件浏览器和脚本命令对话框的设置。

• 控制面板 -用于更改属性面板行为的设置。

• 外观 -在应用程序界面上更改颜色和字体的设置。

• 节点颜色 -用于更改不同节点的颜色的设置。

• 节点图 -用于更改节点图外观的设置 (例如，颜色、字体、背景网格使用、箭头大小和点节点行为)。

• 观众 -用于更改颜色、控件、交互速度和观众缓冲区位深度的设置。

• 脚本编辑器 -用于更改脚本编辑器的行为和语法高亮颜色的设置。

• 范围 -用于更改行为的设置 Nuke 工作室 的范围。

下表描述了可用的首选项设置及其功能。首选项的每个选项卡都在其自己的表中进行了描述。

文件和路径

自动保存文件名

定义在哪里和在什么名字下 Nuke 工作室 保存自动备份文件。默认情况下，文件以扩展名保存 。自动保存 在与项目文件相同的文件夹中。

要更改此设置，请在 自动保存文件名 字段。你可以使用 [值根。名称] 引用完整的脚本路径名，并 [文件尾部 [值根名称]] 仅指带有扩展名的文件名。

空闲后自动保存

定义多长时间 (秒) Nuke 工作室 在系统处于空闲状态 (即没有使用鼠标或键盘) 后，在执行自动备份之前等待。如果将该值设置为 0，将禁用自动备份。

在之后强制自动保存

定义多长时间 (秒) Nuke 工作室 在执行自动备份之前等待，无论系统是否空闲。如果将该值设置为 0，将禁用强制自动备份。

本地文件缓存

本地化到

输入存储所有本地化文件的文件路径。本地化文件允许更快地重新加载存储在访问速度慢的位置 (如网络驱动器) 的文件。

您应该按如下方式构造文件路径

• 用下划线替换任何前导向前或向后斜线。 例如，替换 \ \ Windowspath \ to \ my \ network \ file.dpx 与 _ _ Windowspath \ to \ my \ network \ file.dpx .

• 将任何 Windows 驱动器符号 (冒号) 替换为下划线。例如，替换 T: \ my \ network \ path \ file.dpx 与 T _ \ my \ network \ path \ file.dpx。

自动本地化

输入您需要自动本地化的文件的位置，除非在读取节点中另有指定 缓存 本地 控制。通常这将是您的远程工作文件夹。如果将此字段留空，则不会进行自动本地文件缓存。

记忆

缓存内存使用率 (%)

设置 RAM 的数量 Nuke 工作室 用于处理图像。通常，默认设置在性能和交互性之间进行了很好的权衡。

主动缓存

启用此选项以提高的性能 Nuke 工作室 在具有 8gb 或更多内存的系统上。这应该在内存较少的系统上保持禁用状态。

播放缓存内存 (%)

集总金额 缓存内存使用情况 那 Nuke 工作室 可以用作回放缓存。最近使用的帧保留在内存中，以避免依赖磁盘缓冲系统。

Gpu 纹理缓存内存 (Mb)

设置 GPU 内存量 Nuke 工作室 缓存纹理时使用，以兆字节为单位。默认设置为 256 MB。

磁盘缓存

Nuke 工作室 尝试保存查看器中显示的所有最近图像，以便快速播放。使用此控件，可以指定您想要的位置 Nuke 工作室 保存这些图像。选择本地磁盘 (例如，c:/temp)，最好有最快的访问时间。为最大磁盘缓存大小留出足够的空间也很重要 (定义如下)。

环境变量 NUKE_DISK_CACHE 可用于覆盖此设置。

磁盘缓存大小 (GB)

设置磁盘缓存可以达到的最大大小。确保磁盘上有足够的空间容纳此值。

环境变量 NUKE_DISK_CACHE_GB 可用于覆盖此设置。

绘制磁盘缓存大小 (GB)

设置 RotoPaint 磁盘缓存可以达到的最大大小。确保磁盘上有足够的空间容纳此值。

可用于油漆的 nuke 内存 (%)

将 RotoPaint 节点的内存使用限制为可用的百分比 Nuke 工作室 记忆。

撤销历史记录大小 (MB)

选择用于撤销历史记录的 RAM 量。如果超过此限制，旧项目将被丢弃。

最小撤销事件

选择所需的最小撤销事件数 Nuke 工作室 存储，即使它们违反了内存限制。

GPU 设备

您可以启用 使用 GPU (如果可用) 控制某些节点，使它们在图形处理器 (GPU) 上运行，而不是在中央处理器 (CPU) 上运行。的 GPU 设备 下拉菜单允许您选择要使用的可用设备。您必须重新启动 Nuke 工作室 以便变更生效。

GPU 处理需要某些 NVIDIA GPU 和 CUDA 驱动程序。有关最低要求的更多信息，请参考 Nuke 工作室 中提供的发行说明 帮助 > 发行说明 .

如果平行而不接触

禁用时，窗口捕捉仅限于附近的窗口。

查看器缓冲区位深度

设置默认查看器 OpenGL 缓冲区深度:

• 浮动 -使用全浮动纹理。

• 半浮 -转换为 16 位 (半) 浮动。

• 字节 -使用 8 位错误扩散。这是默认设置。

当 Nuke 工作室 设置为使用 8 位 ( 字节 ) Gl 缓冲区深度 最后，我们在从内部浮点图像数据到 8 位的转换中进行误差扩散。这是为了避免由于位深度减小而出现条带。没有这个，艺术家经常错误地假设他们看到的条带实际上存在于他们的图像数据中，并试图模糊以删除它，不必要地软化图像。然而，因为 Nuke 工作室 在浮点内部工作，实际上几乎没有带状的可能性。

当 Gl 缓冲区深度 设置为 半浮 或 浮动 因此，应用有序抖动，而不是误差扩散。

尽可能为查看器使用 GPU

如果选中此选项，查看器将在可能的情况下在 GPU 中应用其效果 (如 gain 、 gamma 和查看器进程节点)。但是，在某些情况下，如启用监控器输出或 Gl 缓冲区深度 设置为 字节 在查看器设置中，仍然必须在 CPU 中计算效果。

尽可能使用 GPU 进行输入

通常，查看器只尝试在 GPU 上运行自己的效果 (如增益、伽马和查看器进程节点)。然而，当检查这一点时，任何连接到查看器的节点也会尽可能在 GPU 上计算。请注意，不能对所有节点都这样做，因为并非所有节点都有 GPU 实现。

如果在 GPU 上计算节点，查看器中显示的颜色值是不准确的。这是因为它们显示了在将图像传输到显卡之前，CPU 中计算的最后一个节点的颜色。

选中此选项可在使用半浮动深度时禁用查看器中的抖动。取消选中以始终允许抖动。

播放期间优化查看器

Nuke 界面中的逐帧更新可能非常昂贵，因此实时回放可能会受到影响。曲线编辑器、涂料表和 RotoPaint 或跟踪器节点是界面中更新较多区域的好例子。

您可以检查这一点，以便在 RAM 缓存播放期间停止接口上其他地方的更新，从而帮助优化播放帧速率。这将影响以下接口区域，直到播放停止，此时整个接口都将更新:

• 属性面板控件-控件在播放过程中不会更新，它们显示了向前或向后按下播放时您所在帧的值。停止播放时，控件将更新以显示当前帧的值。

• 节点图-自动标签和邮资戳不更新。

• 曲线编辑器/摄影表-黄色时间线指示器被隐藏，直到播放停止，此时它显示在当前帧。

您仍然可以正常操作控件、曲线和关键帧，并检查 播放期间优化查看器 不会影响查看器中的句柄。

当 播放期间优化查看器 未选中，所有接口元素都将根据需要更新。

注意: 此设置仅影响查看器 在新脚本中 .

包括查看器颜色变换

启用时，范围数据包括应用的查看器颜色变换 (增益、伽玛和 LUT)。

禁用时，范围数据不包括应用的查看器颜色转换。这可能会减慢渲染速度，因为它可能需要图像计算。

力全框架

启用后，范围将显示整个框架的数据，而不管该框架的哪一部分在查看器中显示。

禁用时，范围仅显示查看器请求的当前区域的数据，而不是整个框架。
