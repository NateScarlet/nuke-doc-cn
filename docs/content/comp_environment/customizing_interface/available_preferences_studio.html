下降帧

使用此选项可以选择此序列中的时间码是否以放置帧时间显示。

Drop Frame 是一个时间码显示选项，每分钟从 30 fps 时间码序列中删除两个帧 (每 10 分钟除外) 因此，长时间运行的 NTSC 序列精确到实时时钟 (NTSC 帧速率为 3000/1001，或比 30fps 慢约 0.01%)。

红色火箭

使用红色火箭

您可以选择 使用红色火箭 复选框以加快红色媒体的解码速度。如果你正在使用 R3D 火箭显卡，请注意，当你以全分辨率阅读时，使用它可能只会更快。例如，如果你以半分辨率阅读，在没有启用 R3D 火箭卡的情况下使用可能会更快。这是因为 R3D Rocket 显卡在同时读取多个帧时速度很快。这不是它在内部的工作方式，因此当在较低分辨率 (<4k 宽度) 下工作时，禁用 R3D 火箭卡的读取有时可能会更快。此外，请注意，R3D 火箭卡总是比下采样时产生更好的结果。此外，R3D Rocket 卡一次只能由一个应用程序使用，因此如果您同时查看多个脚本, 您可能只能在一个中使用 R3D 火箭卡。

GPU

扩展 3 到 4 个频道

您可以使用此扩展缓存用于播放的图像，每个像素从 3 到 4 个颜色通道。一些图形硬件在每像素 4 个通道将图像加载到视频内存方面比在 3 个通道中加载图像时表现得更好。启用此选项可以提高此类硬件上的播放性能，而代价是减少可以缓存的帧数量。

如果播放性能不佳，启用此选项可能会有所帮助。但是，如果您在禁用此选项的情况下看到可接受的播放性能，则禁用此选项会增加可以缓存以进行平滑播放的帧数量。

注意: 您必须重新启动 Nuke 工作室 此选项将生效。

启用 PBO 纹理上传

渲染

导出渲染

您可以从几个渲染选项中进行选择:

• 限制渲染器 (更敏感的 ui) -选择此选项可使用户界面在转码过程中更具响应性。它告诉 Nuke 工作室 使用 2 个线程进行转码，并使用 25% 的 RAM 进行缓存。使用此选项可能会导致转码速度较慢。

• 没有渲染器限制 -选择此选项以确保转码尽快发生。此选项可能会导致在转码过程中用户界面响应能力下降。

• 自定义渲染限制 -选择此选项以手动配置 Nuke 工作室 的缓存内存设置和用于进行转码的线程数。

线程数

设置线程的数量 Nuke 工作室 转码时使用。较低的数字允许 时间轴环境 的界面更敏感。更高的数字允许更快的转码。此设置传递给 Nuke 工作室 使用 -M 选项。

缓存内存 (GB)

使用此设置千兆字节的 RAM 数 Nuke 工作室 用于其缓存设置。较低的数字可能会改善 时间轴环境 的接口响应，而较高的数字可能会提高转码的速度。此设置传递给 Nuke 工作室 随着 -C 选项。

背景渲染

设置背景渲染发生的时间:

• 不要自动启动背景渲染 -时间轴上的 Comps 不会自动在后台渲染。

• 开始背景呈现在 Comp 保存 -保存时，时间轴上的 Comps 会自动在后台渲染。

• 开始背景呈现在 Comp 创建、 Comp 保存和 Comp 版本更改上 -首次创建、保存和选择新版本时，时间轴上的 Comps 会自动在后台渲染。

字节

使用此选项选择 OpenGL 缓冲区深度，并允许将 GPU 用于查看器进程和输入进程。

• 字节 -转换为 8 位错误扩散。

• 半浮 -转换为 16 位 (半) 浮动。在这种模式下，GPU 可以用于在查看器过程中应用查看器效果，如 gamma 和 LUT。

• 浮动 -使用完整的 32 位浮点纹理。这在选定的卡上可能很慢。在这种模式下，GPU 可以用于在查看器过程中应用查看器效果，如 gamma 和 LUT。

你可以选择一个预设值的设定, 偏好 ,或者在启动脚本中使用 knobDefault ()。

尽可能为查看器使用 GPU

如果选中此选项，查看器将在可能的情况下在 GPU 中应用其效果 (如 gain 、 gamma 和查看器进程节点)。但是，在某些情况下，如启用监控器输出或 Gl 缓冲区深度 设置为 字节 在查看器设置中，仍然必须在 CPU 中计算效果。

尽可能使用 GPU 进行输入

通常，查看器只尝试在 GPU 上运行自己的效果 (如增益、伽马和查看器进程节点)。然而，当检查这一点时，任何连接到查看器的节点也会尽可能在 GPU 上计算。请注意，不能对所有节点都这样做，因为并非所有节点都有 GPU 实现。

如果在 GPU 上计算节点，查看器中显示的颜色值是不准确的。这是因为它们显示了在将图像传输到显卡之前，CPU 中计算的最后一个节点的颜色。

禁用 GPU 查看器抖动

选中此选项可在使用半浮动深度时禁用查看器中的抖动。取消选中以始终允许抖动。

新观众没有不完整的立体声
