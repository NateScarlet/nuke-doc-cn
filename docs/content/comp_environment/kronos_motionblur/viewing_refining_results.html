查看和精炼结果

要查看和细化结果:

1。 从中选择所需的向量计算类型 方法 下拉列表:

• 本地 -使用局部块匹配来估计运动矢量。此方法处理速度更快，但会导致输出中出现瑕疵。

• 正规化 -使用半全局运动估计在区域之间产生更一致的向量。

注意: 从早期版本的 Nuke 默认加载的脚本 本地 向后兼容的运动估计。将新的 VectorGenerator 节点添加到节点图默认为 方法 到 正规化 运动估计。

2. 要查看生成的运动矢量字段，请单击查看器顶部的 “通道” 下拉菜单，然后选择:

• 运动 -查看向前和向后运动向量。

• 向前 -查看向前运动矢量。

• 向后 -查看向后运动向量。

3. 如果计算的运动矢量字段在与其他节点 (如 Kronos 和 MotionBlur) 一起使用时没有产生您想要的结果，请尝试调整 矢量细节 在 VectorGenerator 属性中。

这决定了向量场的分辨率。向量细节越大，处理时间就越大，但向量应该越详细。值 1.0 在每个像素处生成一个向量。值为 0.5 时，会在每个其他像素处生成一个向量。对于某些序列，接近 1.0 的高矢量细节会产生太多不需要的局部运动细节，通常低值更合适。

4. 如果您使用 正规化 矢量计算方法，调整 强度 控件确定帧之间像素匹配的强度。更高的值允许您准确地将一幅图像中的相似像素与另一幅图像进行匹配，即使生成的运动场参差不齐，您也可以集中精力进行细节匹配。较低的值可能会错过本地细节，但不太可能为您提供奇怪的伪向量，从而产生更平滑的结果。

注意: 默认值应该对大多数序列都很有效。

5. 如果您使用 本地 矢量计算方法，调整 平滑度 控制以改善您的结果。高平滑度可能会错过许多局部细节，但不太可能为您提供奇怪的杂散向量，而低平滑度集中在细节匹配上，即使结果场是锯齿状的。

NOTE: The default value should work well for most sequences.

6. 如果源序列中的亮度和总体闪烁有差异，请启用 闪烁补偿 为了避免输出出现问题。

可变亮度的例子包括金属表面的高光，如车身，或以不可预测的方式反射光线的层内的水体。

注意: 启用 闪烁补偿 增加渲染时间。

7. 默认情况下，VectorGenerator 根据图像的亮度分析运动。具体来说，这是红色、绿色和蓝色通道的平均值。但是，您可以使用 重量红色 , 重量绿色 ,和 重量蓝色 控制下 公差 。例如，如果您设置 重量红色 和 重量绿色 参数为零，VectorGenerator 只在蓝色通道中查找运动。
8。 一旦您对结果感到满意，我们建议您在 VectorGenerator 之后插入一个写节点，以将原始图像和矢量通道渲染为 。Exr 文件。这种格式允许存储嵌入多层的图像。以后，无论何时使用相同的图像序列，运动矢量场都会加载到 Nuke 与序列一起。
