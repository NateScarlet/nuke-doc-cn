连接 Kronos

要连接 Kronos:

1。 选择 时间 > Kronos 要在要重定时的序列后插入 Kronos 节点。

的 输入范围 首次创建节点时，由源剪辑自动设置。之后，只有当您单击时才会更新 重置 .

2. 如果你打算使用 输出速度 或 框架 定时，将查看器附加到 Kronos 节点的输出,

OR

如果你打算使用 输入速度 计时，附加查看器 之前 Kronos 节点。

3. 启用 使用 GPU (如果可用) 在上呈现输出 本地 GPU 指定 (如果可用)，而不是 CPU。GPU 和 CPU 之间的输出在 NVIDIA GPU 上是相同的，但是使用 GPU 可以显著提高处理性能。

注意: 如果您的计算机进入睡眠模式，CUDA 驱动程序无法恢复，您必须重新启动 Nuke 才能使用 GPU 加速。

Nuke 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上支持 AMD gpu，运行 OS X 10.9.3 “mavericks” 或更高版本。虽然，在某些情况下，GPU 和 CPU 之间的比特级相等保持不变，但对于某些操作，这种配置可能存在精度限制。

注意: 在没有本地 GPU 的情况下启用此选项允许脚本在 GPU 上运行，只要脚本在具有 GPU 可用的机器上打开。
你应该选择这个，如果你想使从命令行的 -- Gpu 选项。

有关最低要求的更多信息，请参见 GPU 加速要求 或指 Nuke 中提供的发行说明 帮助 > 发行说明 .

4. 选择要应用重定时的通道，使用 通道 下拉列表。Kronos retimes 所有 默认情况下是频道。
5. 如果您的序列由在背景上移动的前景对象组成，则运动估计可能会在两者之间的边缘混淆。要解决此问题，请将前景区域的遮罩添加到 哑光 输入。然后，使用 哑光通道 在 Kronos 属性中选择要使用的遮罩组件。

这将强制前景的运动与背景的运动分开计算，因此在重定时序列中产生的伪影应该更少。

6. 如果输入序列中的运动之前已经估计过 (例如，使用 矢量发生器 或第三方软件)，您可以向背景向量提供运动向量 ( BgVecs ) 和前景向量 ( FgVecs ) 输入以节省处理时间。

注意: 的 BgVecs 输入显示为节点一侧的箭头。

如果背景和前景有单独的向量，您应该将它们连接到适当的输入，并将用于生成它们的遮罩提供给 哑光 输入。如果你有一组向量，你应该把它连接到 FgVecs 输入。

7. 你可以设置 输出 在 Kronos 控件中:

• 结果 -显示前景和背景重定时镜头。

• 哑光 -显示重新计时 哑光 输入。

• 前景 -显示重新计时的前景。遮罩输入之外的背景区域可能显示垃圾。

• 背景 -显示重新计时的背景。哑光输入中的前景区域可能显示垃圾。

8。 如果你 源 序列非常嘈杂，干扰了运动估计，您可以在 MoSrc 输入。当一个 MoSrc 输入连接，运动矢量从该序列计算并应用于 源 序列。
9. 继续到 重新计时序列 .
