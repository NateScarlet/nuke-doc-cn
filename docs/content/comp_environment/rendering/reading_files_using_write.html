使用写入节点读取渲染图像

您可以使用写入节点来渲染图像并将渲染的图像读回。因为从文件中读取写节点的输出比通过上游处理节点树来计算其输出更快，所以这在大型 comps 中特别有用。完成节点树分支的工作后，可以在其后插入写入节点，渲染输出，并使用相同的写入节点读取中渲染的图像。如果您以后需要编辑上游的节点，只需进行更改，并再次渲染写入节点以更新正在读取的图像。

使用写入节点在渲染图像中读取

1。 渲染图像，如中所述 输出 (写) 节点 。我们建议将图像渲染为 。Exr 。这样, 核武器 将传入节点树的哈希值写入渲染文件。如果节点树发生变化，渲染的文件过期，哈希将不匹配 核武器 通知你这个问题。
2. 在 “写入节点” 属性中，选中 读取文件 。当这个打开时, 核武器 忽略上游节点树，并将渲染图像用作写入节点的输出。
3. 要检查输入文件是否是最新的，输入树连接到写入节点，请检查 检查文件匹配输入 。这只适用于 。Exr 写入的文件 核武器 当代理模式和 down-rez 被禁用时。如果无法检查输入文件, 核武器 显示单词 未检查 在节点图中的写入节点上。
4. 如果加载渲染文件时出现错误，请从 缺少帧 下拉菜单:

• 错误 -在任何丢失的帧上显示错误消息。

• 黑色 -用黑色替换任何丢失的帧。

• 棋盘 -用棋盘图像替换任何丢失的帧。

• 读取输入 -在任何丢失的帧上显示输入树的结果，而不是渲染的文件。

提示: 您也可以使用 Precomp 节点 ( 其他 > Precomp ) 将节点树的部分缩减为预渲染图像输入。有关更多信息，请参见 使用 Precomp 节点 .

哈希值是什么？

散列值是一个唯一的数字 (例如，b1c9c0aff2012a8)，从节点和连接到其输入的整个节点树中计算出来。节点的类和所有当前控制设置都有助于哈希值。

通过选择节点图中的节点并按键，可以在节点树中的任何点显示哈希值 我 。树中的不同点的哈希不同。
