渲染输出

核武器 可以在工作站上本地渲染图像 (请参见 输出 (写) 节点 ) 或者可以将其设置为在网络渲染场上渲染图像 (请参见 在外部机器上使用帧服务器 )。在呈现之前，请确保您使用了适当的文件名语法 (请参见 渲染图像的文件名约定 ) 并验证您的项目设置是否选择了正确的输出格式和代理格式 (请参见 渲染分辨率和格式 )。

默认情况下，如果选择渲染帧 15，则会相应地对生成的文件进行编号，例如 image.0015.rgb。但是，您可以通过表达式、指定的开始帧和常数偏移来更改此行为 (请参见 更改渲染帧的编号 )。

有时，您可能希望渲染图像，只是为了读取渲染的图像 (请参见 使用写入节点读取渲染图像 )。因为从文件中读取输出比通过上游处理节点树计算输出更快，所以这可以加快大型项目的速度。

注意: 需要的脚本 核武器 要同时加载大量文件 (例如，通过有数百个读取节点后跟时间模糊)，可能会超过每个进程可用文件句柄的数量，从而在呈现脚本时导致问题。

核武器 它本身在所有系统上最多支持 2048年的文件句柄; 但是，您可能需要增加系统上的文件句柄限制。

在 Mac OS X 上，您可以通过从终端输入以下命令，然后运行来增加 256 的默认限制 (取决于您的版本) 核武器 从同一终端会话:
Ulimit-Sn 2048
