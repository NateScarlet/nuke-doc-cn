渲染图像的文件名约定

“写入” 节点中没有用于指定输出格式的参数。相反，在键入文件名时，格式由前缀或扩展名指示。以下是适当的语法:

<前缀>:/<路径>/<名称>。<帧号变量>。<扩展>

可选的 <前缀>: 可以是任何有效的扩展。 <路径> 要渲染的目录的完整路径名。的 <帧号变量> 通常输入为 #### ,用 # 表示帧号填充到四位数。

您可以通过用任意数量的哈希标记替换 # 来更改填充。例如，两位数的填充将是 # # ,三位数将是 ### ,五位数将是 ##### .

考虑到这些约定，假设您要将名为 “final _ comp_v01” 的图像序列保存为 TIFF16 格式。以下是在写入节点中工作的名称示例:

Tiff16:/< 路径 >/Final_comp_v01。 #### 。Tiff

/< 路径 >/Final_comp_v01。 #### .Tiff16

Tiff16:/< 路径 >/Final_comp_v01。 ####

支持的所有扩展 核武器 可用于前缀。请参阅 附录 C: 文件格式 获取可识别扩展的完整列表。

使用前缀时，它优先于由扩展表示的格式。事实上，前缀使扩展变得不必要。

例如，您可以输入 Exr:/<path>/# 作为文件名，这将创建一个只包含帧号的 OpenEXR 图像序列，填充为三位数。

编写渲染图像的版本

你能写出不同版本的文件使用 Alt 向上 / 下来 箭头键。版本必须按照以下格式编写 核武器 认识他们:

< 路径 >/Final_comp _ V01 . #### 。Tiff

< 路径 >/Final_comp _ V02 . #### 。Tiff

< 路径 >/Final_comp _ V03 . #### 。Tiff
