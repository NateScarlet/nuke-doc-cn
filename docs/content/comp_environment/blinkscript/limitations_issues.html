限制和已知问题

仅限 RGBA

处理目前仅限于 RGBA 通道。要使用其他通道，请在连接到 BlinkScript 节点之前使用洗牌节点将它们洗牌到 RGBA。

GPU

如果执行时间过长，复杂的内核可能会导致 GPU 上的驱动程序超时。GPU 的规格越低，就越有可能发生这种情况。

Nuke 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上支持 AMD GPU，运行 OS X 10.9.3 “mavericks”，或者更高版本，在某些情况下，GPU 和 CPU 之间的比特级平等保持不变, 对于某些操作，此配置的准确性受到限制。

崩溃和无限循环

由于 BlinkScript 节点允许编写和运行任意代码，因此有可能崩溃或锁定 Nuke 。BlinkScript 节点在运行代码之前不会检查代码以确保它是明智的，所以我们建议您使用小心!

已知问题

BlinkScript 的当前表单中存在以下已知问题:

OS X 10.8 上的 OpenCL (山狮)

在 Mac OS X 10.8.2 和 10.8.3 上运行 OpenCL 内核可能不可靠。我们建议更新到 10.8.4 或更高版本。
