GPU 还是 CPU？

只读本地 GPU 字段显示本地安装的显卡 (如果有)。您可以切换的 GPU 和关闭使用 使用 GPU (如果可用) 控制。如果您有多个可用的 gpu，请使用 偏好 > 性能 > 硬件 > 默认闪烁设备 下拉列表。

注意: 选择不同的 GPU 需要重新启动 Nuke 在变更生效之前。

提示: 即使本地没有可用的 GPU，您仍然可以启用 使用 GPU (如果可用) 。然后，当一个可用的时候，BlinkScript 使用 GPU。
你应该能够解决这个，如果你想使从命令行的 -- Gpu 选项。

GPU 和 CPU 之间的输出在 NVIDIA GPU 上是相同的，但是使用 GPU 可以显著提高处理性能。

注意: 如果您的计算机进入睡眠模式，CUDA 驱动程序无法恢复，您必须重新启动 Nuke 才能使用 GPU 加速。

Nuke 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上支持 AMD gpu，运行 OS X 10.9.3 “mavericks” 或更高版本。虽然，在某些情况下，GPU 和 CPU 之间的比特级相等保持不变，但对于某些操作，这种配置可能存在精度限制。

运行 Nuke 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上，运行 OS X 10.9.3 “mavericks” 或更高版本，增加了一个额外的控制，允许您启用多 GPU 处理。该选项是全局的，适用于所有 GPU 加速节点，并在 偏好 > 性能 > 硬件 GPU 下的选项卡。

注意: 的 多 GPU 仅当您使用 Nuke 在 2013年底的 Mac Pro 6.1 上，运行 OS X 10.9.3 “mavericks” 或更高版本。请参阅 GPU 加速要求 有关使用的更多信息。

此外，启用 在 CPU 上矢量化 尽可能在 CPU 上使用 SIMD 加速。请参阅 帮助 > 文档 有关内核执行的更多信息。
