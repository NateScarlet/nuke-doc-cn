挑选屏幕颜色

的 屏幕颜色 可能是键控中最重要的控制，在做其他事情之前，你应该总是选择屏幕颜色。它应该设置为前景对象后面的绿色或蓝色。

1。 开始 Nuke 并读取前景和背景图像。从 Keyer 菜单，选择 ChromaKeyer 并附上观众。
2. 在 ChromaKeyer 属性 面板中，单击旁边的色样 屏幕颜色 激活滴管。
3. 直接从图像中选择屏幕颜色 Ctrl / Cmd + 转变 并在绿色像素上拖动矩形。使用选定像素的平均值。

提示: 您可以通过 Ctrl / Cmd 在查看器中单击鼠标右键。

从屏幕的不同部分选择不同深浅的绿色或蓝色是值得的，以获得最佳效果。

注意: 重复选择颜色不会添加到以前的选择中，每次点击都会添加更多的图像。要键入更多图像，请在下游的其他 ChromaKeyer 节点中选择不同深浅的绿色或蓝色，并设置 内面具 控制至 源 alpha 。请参阅 多程键控 欲了解更多信息。

选择的 屏幕 颜色 做两件事:

• 它创建初始 哑光 用于合成前景下游。磨砂可提高使用 哑光 中的控件 属性 面板。请参阅 改善遮罩 欲了解更多信息。

• 它鄙视前景，但你可以使用 绝望 更高的精度控制。有关更多信息 绝望 ,请参见 绝望和颜色替换 欲了解更多信息。

调整屏幕增益和平衡

的 屏幕增益 控制移除多少屏幕颜色以使哑光。增加此值会更多地显示前景。在示例图像中，较低的 屏幕增益 值增加了太多的背景回到输出，而高 屏幕增益 值会过度侵蚀前景，并倾向于将边缘染成与屏幕颜色相反的颜色 (对于绿色屏幕，边缘会变成洋红色)。

一个 屏幕增益 小于 1 的值将背景颜色添加回图像。

一个 屏幕增益 大于 1 的值将从图像中删除背景色。

的 屏幕平衡 控制两种非基色在 屏幕颜色 已被选中。一个 屏幕平衡 “更接近 0” 使输出朝着两种颜色中强的方向平衡，而 “更接近 1” 的值朝着较弱的颜色。在示例图像中，背景色为绿色，RGB 值为 0.04 、 0.7 和 0.07，因此接近 0 的值会影响蓝色分量。

一个 屏幕平衡 接近 0 影响蓝色成分。

一个 屏幕平衡 接近 1 影响红色成分。
