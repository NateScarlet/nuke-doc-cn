使用 Precomp 节点加快渲染速度

如果您想使用 Precomp 节点来加快渲染速度，您需要在 Precomp 脚本的输出中读取 precomp 节点。

渲染 Precomp 节点

1。 创建 Precomp 节点。有关如何执行此操作的更多信息，请参见 创建 Precomp 节点 .
2. 在 Precomp 节点控件中，单击 渲染 。输入帧范围 (例如, 1-100 ) 在打开并单击的对话框中 好 .

Nuke 呈现 precomp 脚本的输出，并将生成的图像保存在 Precomp 渲染路径 创建 Precomp 节点时指定的目录。

如果已激活代理模式并在输出节点中指定了代理文件名，则渲染的图像是代理图像。

3. 如果渲染成功完成, 读取输出文件 在 Precomp 属性中自动打开。选中此选项后，Precomp 节点变为绿色，并读取渲染的 precomp 图像，而不是计算 precomp 脚本的输出。这个词 (阅读) 读入的图像的名称也显示在节点图中的 Precomp 节点上，以表明这一点。
4. 如果打开预压缩节点的渲染输出时出错，请从 缺少帧 Precomp 属性中的下拉菜单:

• 错误 -在任何丢失的帧上显示错误消息。

• 黑色 -用黑色替换任何丢失的帧。

• 棋盘 -用棋盘图像替换任何丢失的帧。

• 读取输入 -在任何丢失的帧上显示输入树的结果，而不是渲染的文件。

当像这样使用几个 Precomp 节点来替换大型复杂节点树中的部分时，渲染时间可能会变得更快。

注意: 呈现 precomp 脚本的输出时, Nuke 通过首先查找输出节点，自动选择要使用的写入节点。如果它找不到任何输出节点，它会尝试找到一个写节点，并设置 输出节点 在 Precomp 节点中，控制写入节点的名称。如果找不到任何输出或写入节点，它会产生错误。
在任何时候，Precomp 节点属性都可以用来覆盖预压缩脚本中设置为输出节点的内容。为此，请打开 Precomp 属性和 高级 控制。在 输出节点 字段中，输入要使用其输出的写入节点的名称。确保你也检查 启用 。此复选框允许您在默认情况下选择的输出节点和在 输出节点 字段。

提示: 您还可以以类似的方式使用写节点，并将其输出从渲染文件中读取，而不是使用上游节点计算。 有关更多信息，请参见 使用写入节点读取渲染图像 在 预览和渲染 第一章 Nuke 用户指南。
