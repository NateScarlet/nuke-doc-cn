协作工作流示例

在本例中，合成器负责交付给客户端的主 comp。照明 TD 负责提供多层 CG 渲染，用于主 comp。下面描述了合成器和照明 TD 如何使用 Precomp 节点来启用协作工作流。

启用协作工作流

1。 合成器开始构建主 comp。
2. 照明 TD 对 CG 元素进行渲染，并对图层进行比较，将脚本保存为 cg_v01.nk。

警告: 请注意，我们建议将 precomp 图像渲染为 。Exr 文件。这样，Nuke 将传入节点树的哈希值写入渲染文件。如果 precomp 脚本更改为哈希值不匹配，Nuke 可以通知用户，然后用户可以更新生成的图像。如果您使用的文件格式不是 。Exr ,您不会收到此通知，渲染的文件可能会过时。
有关哈希值的更多信息，请参见 创建 Precomp 节点 .

3. 合成器创建从文件 cg_v01.nk 读取的 Precomp 节点，并继续处理主 comp。
4. 照明 TD 继续修改 CG 渲染和 comp，对写节点和 precomp 脚本进行版本化。Nk 名称。(见 Precomp 修订 )。

当获得更好的结果时，照明 TD 会通知合成器 precomp 脚本的新版本和改进版本。

5. 合成器将 Precomp 节点上的版本升级到新的改进版本，并重新加载 precomp 脚本。(见 Precomp 修订 )。
