使用 Precomp 节点

Precomp 节点类似于组节点，但其内容存储在独立的 。Nk 文件。这允许您将节点树的子集保存为单独的 。Nk 脚本，渲染这个保存的脚本的输出，并将渲染的输出作为单个图像输入读回主 comp。

注意: 在 Nuke Assist 中不支持 Precomp 节点。您不能在 Nuke Assist 中创建 Precomp; 但是，如果脚本中已经存在一个 Precomp，您可以查看，但不能修改其输出。为了表明这一点，Precomp 节点在节点图中以红色概述，并且它们的控件是灰色的。请参阅 Nuke 产品 欲了解更多信息。

Precomp 节点至少有两种有用的方式。首先，它们可以用来将节点树的一部分减少到预渲染的图像输入。这加快了渲染时间，因为 Nuke 只需处理单个图像输入，而不是所有用于创建它的节点。因为原始节点保存在单独的 。Nk 脚本，您还可以保持对它们的访问，如果需要，可以在以后调整它们。

其次，Precomp 节点支持协作工作流。当一个艺术家在主 comp 上工作时，其他艺术家可以在使用 Precomp 节点导出的部分上工作。这些部分可以独立于主 comp 进行编辑、版本控制和管理。例如，假设你有一个包含复杂的多层 CG 渲染的 comp。3D 艺术家可以制作一个单独的脚本，排字师完成拍摄，然后使用 Precomp 节点读取。这样，3D 艺术家可以修改和重新渲染 comp 的 CG 元素部分，而不必与合成器共享主 comp。
