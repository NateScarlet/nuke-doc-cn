使用脚本分析

核武器 可以在屏幕上显示准确的性能计时数据，或将其输出到 XML 文件，以帮助您解决慢速脚本中的瓶颈问题。当启用性能计时时，计时信息显示在节点图中，节点本身根据每个节点花费的总处理时间的比例进行着色, 从绿色 (快速节点) 到红色 (慢速节点)。

注意: 您还可以使用 Python 访问单个节点的计时信息。看到的 Python 开发指南 , 性能分析 部分了解更多信息，或单击 在这里 .

启用性能计时

可以从脚本编辑器或命令行使用 -P 参数。你也可以输出的同步数据到 XML 文件中加上 -Pf 参数和文件名，但输出仅在渲染时生成。

注意: 以交互方式或从命令行启用性能计时会增加额外的处理开销，但数据仍然提供了脚本中处理时间的准确位置。

在屏幕上输出性能时间

启用屏幕上的性能计时可以让您在工作时查看每个节点的累积数据。

在脚本编辑器中，输入:

Nuke.startPerformanceTimers ()

在命令行中，输入:

。/Nuke <版本>.exe-P

注意: 您可以使用重置计时数据 Nuke.resetPerformanceTimers () 从脚本编辑器。

当性能计时器处于活动状态时，屏幕上会显示以下信息:

• Cpu -CPU 在所有 CPU 线程上执行处理代码所花费的时间，以微秒为单位。例如，对于多线程处理，这通常比 墙 时间。如果每个线程的平均 CPU 时间 ( Cpu 除以使用的线程数) 比 墙 时间，这表明 CPU 线程花了很多时间不执行代码，也许等待锁，这可能表明性能问题。

注意: 在 Mac OS X 和 Windows 上，CPU 时间目前不准确。在 Mac OS X 上, Cpu 值总是与 墙 时间。

• 墙 -由墙上的时钟测量的时间-你必须等待处理完成的实际时间。的 墙 时间也以微秒为单位。

• Ops -节点中调用的运算符数。运营商是 核武器 执行某些任务的构建块。节点可以包含一个或多个 ops。例如，当一个节点需要调整大小时，它会使用转换操作，而不是自己的实现来做同样的事情。

看到的 NDK 开发指南 , 术语 部分了解更多信息，或单击 在这里 .

• 记忆 -节点使用的系统内存总量。

除了时间信息之外，节点还根据其轮廓进行颜色编码，从绿色到红色，其中红色是一个慢节点。您可以从示例脚本中看到，散焦是一个缓慢的节点，而合并根本不起作用。

注意: 您可以使用停止显示计时数据 Nuke.stopPerformanceTimers () 在脚本编辑器中。

将性能计时输出到文件

你可以输出的同步数据的 XML 文件, -Pf 参数和文件名。输出仅在渲染时生成，无论是从写节点还是命令行渲染。

例如，在 Windows 上，当从节点图中的写入节点进行渲染时，以下命令会将配置文件数据写入文件:

Nuke <version> \ Nuke <version>.exe-Pf <文件路径和名称>.xml

从命令行渲染，以下命令从名为 ProfileTest.nk 并将配置文件数据写入 XML 文件:

核弹 <版本> \ 核弹 <版本>。exe-x-Pf C: \ temp \ profileest.xml C: \ temp \ profileest.nk 1-10
