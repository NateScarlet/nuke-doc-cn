OFlow 重新计时

OFlow 节点生成高质量的重定时操作，分析帧中所有像素的移动，然后基于该分析渲染新的 “中间” 图像。此节点还可以添加运动模糊或增强图像中的现有运动模糊。

提示: 我们建议使用 NukeX 的 Kronos 节点，因为它包括一个改进的运动估计算法。有关更多信息，请参见 Kronos .

用 OFlow 重新计时

1。 选择 时间 > OFlow 在要重定时的序列后插入 OFlow 节点。

3. 默认情况下，OFlow 设置为执行线性半速度减速，使用 计时 > 输出速度 .
4. 启用 使用 GPU (如果可用) 在上呈现输出 本地 GPU 指定 (如果可用)，而不是 CPU。GPU 和 CPU 之间的输出在 NVIDIA GPU 上是相同的，但是使用 GPU 可以显著提高处理性能。

注意: 如果您的计算机进入睡眠模式，CUDA 驱动程序无法恢复，您必须重新启动 Nuke 才能使用 GPU 加速。

Nuke 在 2013年底的 Mac Pro 6,1 和 2015年中的 Mac Pro 11,5 上支持 AMD gpu，运行 OS X 10.9.3 “mavericks” 或更高版本。虽然，在某些情况下，GPU 和 CPU 之间的比特级相等保持不变，但对于某些操作，这种配置可能存在精度限制。

注意: 在没有本地 GPU 的情况下启用此选项允许脚本在 GPU 上运行，只要脚本在具有 GPU 可用的机器上打开。
你应该选择这个，如果你想使从命令行的 -- Gpu 选项。

有关最低要求的更多信息，请参见 GPU 加速要求 或指 Nuke 中提供的发行说明 帮助 > 发行说明 .

5. 选择要应用重定时的通道，使用 渠道 下拉列表。OFlow retimes 所有 默认情况下是频道。
6. 要调整减速或加速序列，请为 输出 或 输入速度 控制。低于 1 的值会减慢剪辑的速度。1 以上的值加快运动。默认值 0.5 创建了半速减速。四分之一的速度是 0.25。
7. 或者，你可以用 “在输出剪辑的 100 帧，我想看到源剪辑的 50 帧” 来描述重新计时。要这样做，设置 计时 到 源帧 。转到时间线中的帧，并设置 框架 到要显示在该输出位置的输入帧。您需要为此设置至少两个关键帧，以重新计时剪辑。例如，要将 50 帧剪辑减慢一半，您可以设置 框架 到 1 帧 1，到 50 帧 100。做四次减速，设置 框架 至 1 帧 1，至 25 帧 100。
8。 如果你想在你的重定时镜头中添加运动模糊，请继续 添加运动模糊 .

提示: 一些 Nuke 用户可能会记住名为 OpticalFlow 和 OptiFlow 的节点。OFlow 替换这些节点。它可以用于重新计时和添加运动模糊，但它不具备输出运动矢量的能力。要输出运动矢量，可以使用 矢量发生器 (包括在 NukeX 和 Nuke )。

对于重定时和 motionblur，我们建议使用 Kronos 节点而不是 OFlow。有关更多信息，请参见 Kronos .

4. 向量场通常有两个重要的特性: 它们应该准确地将一幅图像中的相似像素匹配到另一幅图像中，并且应该是平滑的而不是嘈杂的。通常，有必要将这些品质中的一个与另一个交换。一个高 平滑度 错过了许多本地细节，但不太可能为您提供奇怪的虚假向量。一个低 平滑度 专注于细节匹配，即使结果字段参差不齐。默认值 1.5 应该对大多数序列都适用。

5. 设置重新计时时应用的重新采样类型:

• 双线性 -默认过滤器。处理速度更快，但在更高的缩放级别下会产生较差的结果。你可以使用 双线性 在使用其他重采样类型之一生成输出之前预览重定时。

• Lanczos4 和 Lanczos6 -这些过滤器有利于缩小规模，并提供一些图像锐化，但需要更长的时间来处理。

6. 如果你的整体亮度 源 帧之间的镜头变化，启用 闪烁补偿 。这允许 OFlow 考虑亮度和整体闪烁的变化，否则可能会导致输出问题。

可变亮度的例子包括金属表面的高光，如车身，或以不可预测的方式反射光线的水体。

请注意，使用 闪烁补偿 增加渲染时间。

7. 为了减少处理时间，大部分运动估计只在亮度上进行 -- 也就是说，使用单色图像。在大多数情况下，这是完全可以接受的，但是 公差 组允许您通过向单个颜色添加偏见来专注于图像中的特定特征。例如，你可能希望增加 重量红色 为了让算法集中精力让主要红色物体的运动正确，以射击中其余物体为代价。
